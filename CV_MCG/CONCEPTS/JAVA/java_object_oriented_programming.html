Java Object-Oriented Programming (OOP) FAQ

1. Explain the principles of OOP: Encapsulation, Inheritance, Polymorphism, and Abstraction.

The core principles of OOP are:

  • Encapsulation: Encapsulation restricts access to certain components of an object to protect its state and behavior.
  • Inheritance: Inheritance allows one class (child) to inherit properties and methods from another class (parent), promoting code reuse.
  • Polymorphism: Polymorphism enables objects to be treated as instances of their parent class, allowing for flexibility in method execution.
  • Abstraction: Abstraction hides complex implementation details and provides a simplified interface to interact with objects.

2. What is the difference between a class and an object in Java?

A class is a blueprint that defines properties (attributes) and behaviors (methods) of objects, while an object is an instance of a class created in memory that holds specific values for its attributes.

3. What is an abstract class, and when would you use it instead of an interface?

An abstract class can have both abstract methods (without implementation) and concrete methods (with implementation). It is used when classes share a common base but have unique implementations. Use it over an interface when default implementations or state are needed.

4. What is the difference between method overloading and method overriding? When would you use each?

  • Method Overloading: Overloading occurs when methods in the same class have the same name but different parameter lists. It is useful for creating variations of a method with different input requirements.
  • Method Overriding: Overriding occurs when a subclass provides a specific implementation of a method already defined in its superclass. Use it to provide specific behavior in the subclass.

5. How do you implement polymorphism in Java? Provide an example.

Polymorphism in Java is implemented through method overriding and interfaces. Example: 

interface Animal {
    void makeSound();
}

class Dog implements Animal {
    public void makeSound() { System.out.println("Bark"); }
}

class Cat implements Animal {
    public void makeSound() { System.out.println("Meow"); }
}

Animal animal = new Dog();
animal.makeSound();  // Output: Bark

6. What are access modifiers (public, private, protected)? When is it appropriate to use them?

  • Public: Accessible from any class.
  • Private: Accessible only within the declared class. Used for hiding implementation details.
  • Protected: Accessible within the same package or by subclasses. Useful for allowing inheritance with restricted access.

7. How message passing over methods works in Java (pass-by-value, pass-by-reference)

Java uses pass-by-value. For primitives, the value is passed, while for objects, the reference to the object is passed by value. This means changes to object fields are reflected outside the method, but reassigning the object itself does not affect the original reference.

8. Do you consider that returning null from a method is good practice?

Returning null from a method can lead to NullPointerException if not handled properly. It's often better to return Optional (Java 8+) or empty collections instead of null to avoid errors.

OOP Interview Questions & Answers

Object-Oriented Programming (OOP) Interview Questions & Answers

1. What is a constructor in Java?

A constructor in Java is a block of code used to initialize an object when it is created. It has the same name as the class and does not have a return type.

2. What is the purpose of the `super` keyword in Java?

The `super` keyword is used to refer to the superclass of the current object. It can be used to call the superclass constructor or methods, or to access fields.

3. What is the difference between `==` and `.equals()` in Java?

`==` compares the memory reference (whether two references point to the same object in memory), while `.equals()` compares the actual contents of two objects.

4. What is the difference between `String` and `StringBuilder` in Java?

`String` is immutable, meaning once created, its value cannot be changed. `StringBuilder` is mutable, allowing modification of the string value.

5. What is the `final` keyword in Java?

The `final` keyword in Java can be used to define constants, prevent method overriding, or prevent inheritance of a class.

6. Explain method overriding in Java.

Method overriding occurs when a subclass provides a specific implementation for a method that is already defined in its superclass.

7. What is the difference between an interface and an abstract class in Java?

An interface can only declare abstract methods (until Java 8, when default methods were introduced), and a class can implement multiple interfaces. An abstract class can have both abstract and concrete methods, and a class can inherit only one abstract class.

8. What is a singleton class in Java?

A singleton class is a design pattern that restricts the instantiation of a class to one object. It ensures that only one instance of the class is created during the application lifecycle.

9. What are the differences between `ArrayList` and `LinkedList` in Java?

`ArrayList` uses a dynamic array for storage, allowing for faster random access but slower insertions and deletions. `LinkedList` uses a doubly linked list, allowing for faster insertions and deletions but slower random access.

10. What are abstract methods in Java?

Abstract methods are methods that are declared without an implementation in an abstract class or interface. They must be implemented by subclasses of the abstract class or by a class implementing the interface.

11. What is the role of the `this` keyword in Java?

The `this` keyword refers to the current instance of the class, helping differentiate instance variables from local variables or parameters.

12. What is a `package` in Java?

A package in Java is a namespace that organizes classes and interfaces, allowing for easier code management and avoiding naming conflicts.

13. What is the difference between `throw` and `throws` in Java?

`throw` is used to explicitly throw an exception, while `throws` is used in a method signature to declare that a method may throw one or more exceptions.

14. What is a static method in Java?

A static method belongs to the class rather than instances of the class. It can be called without creating an object of the class.

15. Explain the difference between `interface` and `abstract class`.

An `interface` defines abstract methods that must be implemented by a class, while an `abstract class` can have both abstract and concrete methods. An interface can be implemented by multiple classes, but a class can inherit only one abstract class.

16. What is polymorphism in Java?

Polymorphism allows an object to take many forms. In Java, it is typically achieved through method overriding (runtime polymorphism) and method overloading (compile-time polymorphism).

17. What are the types of polymorphism in Java?

There are two types of polymorphism in Java: compile-time (method overloading) and runtime (method overriding).

18. How does garbage collection work in Java?

Garbage collection is the process of automatically reclaiming memory that is no longer in use, performed by the Java Virtual Machine (JVM). It frees memory used by unreachable objects.

19. What is the purpose of the `transient` keyword in Java?

The `transient` keyword is used to indicate that a field should not be serialized. When an object is serialized, transient fields are not included in the serialization process.

20. What is the purpose of the `volatile` keyword in Java?

The `volatile` keyword is used to indicate that a variableâ€™s value will be changed by multiple threads. It ensures that changes to the variable are visible to all threads.

21. What is the difference between `equals()` and `hashCode()` methods in Java?

The `equals()` method is used to compare the contents of two objects, while `hashCode()` returns an integer hash code that represents the object in a hash-based collection like `HashMap`.

22. What is the difference between `synchronized` and `concurrent` collections in Java?

The `synchronized` keyword is used to make methods or blocks of code thread-safe, while `concurrent` collections (e.g., `ConcurrentHashMap`) are designed to provide better concurrency performance without the need for explicit synchronization.

23. How can you prevent inheritance in Java?

In Java, inheritance can be prevented by using the `final` keyword on a class. A `final` class cannot be extended.

24. What is the difference between `List`, `Set`, and `Map` in Java?

A `List` is an ordered collection that allows duplicates, a `Set` is an unordered collection that does not allow duplicates, and a `Map` is a collection of key-value pairs.

25. What is a `default` method in Java interfaces?

A `default` method is a method in an interface with a default implementation. It allows the addition of methods to interfaces without breaking existing implementations.

26. What is method overloading?

Method overloading occurs when multiple methods in a class have the same name but different parameters (number, type, or order of parameters).

27. What is the difference between a `static` and a `non-static` method?

A `static` method belongs to the class and can be called without an object, while a `non-static` method belongs to an instance of the class and requires an object to be called.

28. What is the purpose of the `instanceof` operator in Java?

The `instanceof` operator is used to check if an object is an instance of a particular class or implements a particular interface.

29. What are the benefits of OOP?

Benefits of OOP include modularity, reusability, scalability, maintainability, and the ability to create complex systems with easily understandable components.

30. Can an abstract class be instantiated in Java?

No, an abstract class cannot be instantiated directly. It must be subclassed, and the subclass must provide implementations for all abstract methods.
