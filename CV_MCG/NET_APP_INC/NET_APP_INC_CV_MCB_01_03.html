🔗 “I developed custom API connectors to retrieve, update, and manage customer interactions, leveraging AI-based insights for personalization” — Here’s how I technically did it and how AI fits in:

1. 🔧 Custom API Connectors (Built in Python & Java)

“I created robust API connectors in Python (FastAPI) and Java (Spring Boot) to interface with CRMs (like HubSpot), messaging platforms (like ManyChat), email systems, or internal services.”

Example:

  • GET /interactions?userId=123: Retrieves all conversations.

  • POST /interactions/insights: Submits AI-processed insights to the CRM.

  • PUT /interactions/123/update: Updates contact info or ticket status.

These connectors supported real-time workflows and background sync jobs via schedulers (e.g., Celery or Spring’s @Scheduled).

2. 🧠 AI-Based Insights Generation (Backend-Driven)

“Once data was pulled via the connectors, I used AI models to analyze and extract actionable insights, including:

  • Sentiment Analysis: Detect whether a user message was positive, neutral, or negative using spaCy, Vader, or HuggingFace Transformers.

  • Intent Classification: Trained an NLP model to classify interactions (e.g., "request refund", "technical support", "price inquiry").

  • Engagement Scoring: ML model scored each interaction’s depth and quality, feeding into recommendation or escalation logic.”

“I implemented logging and feedback endpoints where:

  • Retrained models using the accepted vs rejected recommendations
