🧪 Unit Testing with Jest and React Testing Library

💬 What to Say in the Interview

"In my React projects, I employed **Jest** as the testing framework and **React Testing Library** for testing React components. The goal was to ensure that the components behaved as expected, with a focus on testing the user interactions and rendering logic, rather than the internal implementation details."

🔧 How I Implemented Unit Tests

I implemented unit tests for individual React components by focusing on user behavior (e.g., clicking buttons, entering text). This ensured that the components were tested from the user's perspective and not just their internal structure.

Installation

npm install --save-dev jest @testing-library/react @testing-library/jest-dom

Example Test Case for a Button Component

import { render, screen, fireEvent } from '@testing-library/react';
import '@testing-library/jest-dom';
import Button from './Button';

test('Button should call onClick when clicked', () => {
  const handleClick = jest.fn();
  render(<Button onClick={handleClick} label="Click me" />);

  fireEvent.click(screen.getByText(/Click me/i));
  expect(handleClick).toHaveBeenCalledTimes(1);
});

🎨 Focus on User Interaction Testing

I made sure to focus on testing the UI components as the user would interact with them. This ensured that the application was reliable and bug-free from the user's perspective.

Example: Testing Form Input

test('Form input updates state', () => {
  render(<Form />);
  const input = screen.getByLabelText('Name');
  fireEvent.change(input, { target: { value: 'John Doe' } });
  expect(input.value).toBe('John Doe');
});

🔍 Mocking and Assertions

I used **jest.fn()** to mock functions, ensuring that the components correctly triggered events and interacted with external data, such as API calls or external services. This allowed me to validate the logic of components without needing to rely on actual implementations.

Mocking an API Call

jest.mock('axios');
import axios from 'axios';
import { render, screen, waitFor } from '@testing-library/react';
import DataFetchingComponent from './DataFetchingComponent';

test('fetches and displays data', async () => {
  axios.get.mockResolvedValue({ data: 'some data' });
  render(<DataFetchingComponent />);

  await waitFor(() => screen.getByText('some data'));
  expect(screen.getByText('some data')).toBeInTheDocument();
});

🎯 Benefits of Unit Testing in React

Feature Benefit
Focused Testing Tests components in isolation, focusing on user behavior and interaction
Faster Feedback Quick test runs during development to catch issues early
Prevents Regression Tests ensure that new changes do not break existing functionality
Improved Code Quality Writing tests enforces writing cleaner and more modular code
Confidence in Refactoring Provides a safety net when refactoring or adding new features
