🤖 AI Agents I Implemented – What & How

1. Product Recommendation Agent

Trained a collaborative filtering model (using Python & scikit-learn) based on purchase history and customer profiles. This model was deployed as a FastAPI service, consumed by chatbot and frontend systems. 

# FastAPI route serving recommendations
@app.get("/recommendations/{user_id}")
def get_recommendations(user_id: str):
    vector = user_vectorizer.transform(get_user_data(user_id))
    result = recommendation_model.predict(vector)
    return {"recommended_products": result}

Used by ManyChat chatbot to personalize product suggestions during interactions.

2. Financial Forecasting Agent

Built time-series models (ARIMA, XGBoost) to predict future revenue or cash flow. These models were exposed via a secured REST API (Spring Boot + OAuth2). 

// Java Spring Boot Controller
@GetMapping("/forecast/{companyId}")
public ResponseEntity<Forecast> getForecast(@PathVariable String companyId) {
    Forecast forecast = modelService.forecast(companyId);
    return ResponseEntity.ok(forecast);
}

Used by dashboard components and back-office analytics to guide investment decisions.

3. Anomaly Detection Agent

Deployed unsupervised models (Isolation Forest) that flagged abnormal transactions in real time. Python backend (Flask) scanned batches of transaction data and reported scores via API. 

# Flask API
@app.post("/anomaly-score")
def anomaly_score(data: List[Transaction]):
    features = extract_features(data)
    scores = anomaly_model.decision_function(features)
    return {"scores": scores}

Enabled fraud detection modules to trigger alerts or secondary reviews.

4. Support Escalation Agent

Combined a rules engine with ML classification to detect high-priority customer complaints (based on sentiment, intent, and keywords). Used by ManyChat and backend support ticketing logic. 

# Rules + ML classifier
if intent == "complaint" and sentiment_score < -0.5:
    escalate_ticket(user_id, category="urgent")

Improved customer service by escalating only the most critical cases automatically.

5. Token-Aware AI Access Agent

Some agents (forecasting bots, anomaly services) acted autonomously and authenticated via OAuth2 Client Credentials flow to access secured APIs. 

POST /oauth/token
grant_type=client_credentials
client_id=agent-xyz
client_secret=secret

Ensured secure machine-to-machine interactions within the system.

Summary

These AI agents were modular, secure, and integrated into real-time systems — helping automate decisions, improve predictions, and personalize user experiences in financial and customer-facing domains.
