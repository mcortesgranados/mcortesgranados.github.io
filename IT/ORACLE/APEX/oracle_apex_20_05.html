Oracle Application Express (APEX) Questions and Answers - Part 5

81. How do you enable session state protection in APEX?

Session state protection in APEX ensures that URL tampering does not alter the session state of an application. To enable session state protection:

  1. Go to "Shared Components" > "Security" > "Session State Protection".
  2. Enable the protection and configure it for specific pages or items.

82. How do you perform database updates using a form in APEX?

In APEX, to update the database using a form:

  1. Create a form using the wizard.
  2. Define the process to handle the database update after the form submission.
  3. Use a PL/SQL process or Automatic DML handling in Page Designer.

    BEGIN
        UPDATE my_table
        SET column1 = :P1_ITEM1,
            column2 = :P1_ITEM2
        WHERE id = :P1_ID;
    END;

83. What are Authorization Schemes in APEX?

Authorization Schemes in APEX define the rules to control access to certain application components based on user roles or conditions. You can create Authorization Schemes in "Shared Components" and apply them to pages, regions, buttons, etc.

84. How do you create a LOV (List of Values) in APEX?

To create a LOV:

  1. Navigate to "Shared Components" > "List of Values".
  2. Choose "Dynamic" (SQL query) or "Static" LOV, depending on your requirement.
  3. Assign the LOV to form items like select lists, radio buttons, or checkboxes.

    SELECT emp_id, emp_name
    FROM employees;

85. How do you implement conditional logic in APEX pages?

You can implement conditional logic for rendering or processing in APEX by:

  • Using the "Conditions" attribute in Page Designer for items, regions, and buttons.
  • Writing PL/SQL expressions for custom conditions, such as RETURN :P1_ITEM = 'VALUE';.

86. How do you manage session state in APEX?

Session state in APEX stores the values of page items during a session. You can manage session state using:

  • Built-in PL/SQL functions like APEX_UTIL.SET_SESSION_STATE and APEX_UTIL.GET_SESSION_STATE.
  • Using Dynamic Actions to set or reset page items' values.

87. How do you use the "Reset Pagination" feature in APEX?

The "Reset Pagination" feature resets the page number to the first page after an action like a search or report filtering. You can enable this by configuring the "Reset Pagination" option in the report attributes.

88. How do you use dynamic SQL in APEX for interactive reports?

Dynamic SQL can be used in Interactive Reports by referencing PL/SQL code. Here's an example:


    SELECT column1, column2
    FROM my_table
    WHERE column3 = :P1_FILTER;

89. How do you create a tabular form in APEX?

To create a tabular form:

  1. Use the "Create Page" wizard and select "Tabular Form".
  2. Specify the table or SQL query that will be used to populate the form.
  3. The resulting form allows for editing multiple rows in a grid format.

90. How can you send email notifications in APEX?

To send emails in APEX, you can use the APEX_MAIL package:


    BEGIN
        APEX_MAIL.SEND(
            p_to => 'recipient@example.com',
            p_from => 'sender@example.com',
            p_body => 'This is the email body',
            p_subj => 'Email Subject');
        COMMIT;
    END;

91. How do you implement cascading LOVs in APEX?

Cascading LOVs are dependent select lists where the values of one select list depend on the selection made in another. To implement this:

  1. Define a dynamic LOV for both items.
  2. Set the "Cascading LOV Parent Item(s)" attribute for the child LOV.
  3. Use a SQL query that filters based on the parent itemâ€™s value.

92. How do you track user activity in APEX applications?

To track user activity in APEX:

  • Use the "Activity Monitoring" feature under "App Builder" > "Utilities".
  • Log user activity manually using custom logging tables and triggers on specific page events.

93. How do you display a custom success message after form submission?

After a successful form submission, you can display a custom success message using the apex_application.g_print_success_message API:


    apex_application.g_print_success_message := 'Your data has been successfully saved!';

94. What is the difference between "Shared Components" and "Application Items"?

"Shared Components" are reusable components that can be used throughout the application, such as LOVs, templates, and Authorization Schemes. "Application Items" are global variables that hold values that can be referenced across different pages.

95. How do you implement a search feature in APEX?

To implement a search feature in APEX:

  • Create a text field for the search input.
  • Create an Interactive Report that uses the search input in its SQL query, for example:

    SELECT * FROM my_table WHERE column LIKE '%' || :P1_SEARCH || '%';

96. How do you format numbers in APEX forms and reports?

To format numbers in APEX:

  • For forms, set the "Number Format" property in the item's attributes.
  • For reports, use the TO_CHAR function in SQL queries to format numbers:

    SELECT TO_CHAR(salary, '999,999.00') AS formatted_salary
    FROM employees;

97. How do you handle error messages in APEX?

In APEX, you can handle errors using PL/SQL error-handling procedures or built-in validation. For custom error handling, use the apex_error package:


    apex_error.add_error (
        p_message => 'Custom error message',
        p_display_location => apex_error.c_inline_in_notification);

98. How do you create and use a collection in APEX?

APEX collections are used to store data temporarily for the session. You can create and manipulate collections using the APEX_COLLECTION package:


    BEGIN
        APEX_COLLECTION.CREATE_COLLECTION('MY_COLLECTION');
        APEX_COLLECTION.ADD_MEMBER('MY_COLLECTION', p_c001 => 'Value1');
    END;

99. How do you handle file uploads in APEX?

APEX provides built-in support for file uploads through form items. You can also use the APEX_APPLICATION_TEMP_FILES table to handle uploaded files:


    SELECT * FROM apex_application_temp_files;

100. How do you use page redirects in APEX?

To redirect a page in APEX, use the "Branch" component or a PL/SQL procedure:


    apex_util.redirect_url('f?p=&APP_ID.:2:&SESSION.');
