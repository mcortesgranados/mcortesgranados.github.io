Set 1

  1. What is the purpose of the `Executor` framework in Java?

    Answer: The `Executor` framework simplifies the management of threads and provides an abstraction for thread management. It decouples task submission from the details of how each task will be executed, typically using a pool of worker threads.

  2. What is the difference between `ExecutorService` and `Executor`?

    Answer: `Executor` is a simple interface for executing tasks asynchronously, whereas `ExecutorService` is a more specialized interface that includes methods for managing and controlling the lifecycle of tasks, such as `shutdown()` and `submit()`.

  3. What is the purpose of `ForkJoinPool` in Java?

    Answer: `ForkJoinPool` is designed for tasks that can be recursively split into smaller tasks, allowing for parallel execution. It is optimized for applications that perform many small tasks and need to work efficiently with the underlying hardware.

  4. How do you submit a task to an `ExecutorService`?

    Answer: You can submit a task to an `ExecutorService` using the `submit()` method, which returns a `Future` object that can be used to retrieve the result of the task.

    import java.util.concurrent.*;

public class Main {
    public static void main(String[] args) throws InterruptedException, ExecutionException {
        ExecutorService executor = Executors.newFixedThreadPool(2);
        Future future = executor.submit(() -> "Hello, World!");
        System.out.println(future.get());  // Prints: Hello, World!
        executor.shutdown();
    }
}
  5. What is the difference between `synchronized` and `Lock` in Java?

    Answer:

    • `synchronized`: A keyword that allows a method or block to be executed by only one thread at a time.
    • `Lock`: A more flexible and advanced interface that allows for explicit locking and unlocking, offering greater control over the synchronization process.

  6. What is the purpose of `ReentrantLock`?

    Answer: `ReentrantLock` is an implementation of the `Lock` interface that allows a thread to re-enter a lock it already holds. It provides additional methods like `tryLock()` and `lockInterruptibly()` for better control over locking.

    import java.util.concurrent.locks.*;

public class Main {
    public static void main(String[] args) {
        Lock lock = new ReentrantLock();
        lock.lock();
        try {
            // critical section
            System.out.println("Inside critical section");
        } finally {
            lock.unlock();
        }
    }
}
  7. What is `CountDownLatch` used for in multithreading?

    Answer: `CountDownLatch` is a synchronization aid that allows one or more threads to wait until a set of operations being performed by other threads completes. It is initialized with a count and threads can call `countDown()` to decrement it.

    import java.util.concurrent.*;

public class Main {
    public static void main(String[] args) throws InterruptedException {
        CountDownLatch latch = new CountDownLatch(2);
        
        Runnable task = () -> {
            try {
                System.out.println("Task started");
                Thread.sleep(1000);
                latch.countDown();
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        };

        new Thread(task).start();
        new Thread(task).start();
        
        latch.await(); // Main thread waits until count reaches 0
        System.out.println("All tasks completed");
    }
}
  8. What does the `ExecutorCompletionService` do?

    Answer: `ExecutorCompletionService` is used to manage a pool of tasks and allows you to retrieve the results of completed tasks in the order they finish, rather than the order they were submitted.

    import java.util.concurrent.*;

public class Main {
    public static void main(String[] args) throws InterruptedException, ExecutionException {
        ExecutorService executor = Executors.newFixedThreadPool(2);
        ExecutorCompletionService service = new ExecutorCompletionService<>(executor);
        
        service.submit(() -> "Task 1 completed");
        service.submit(() -> "Task 2 completed");
        
        System.out.println(service.take().get());  // Prints: Task 1 completed
        System.out.println(service.take().get());  // Prints: Task 2 completed
        executor.shutdown();
    }
}
  9. What is the purpose of `Semaphore` in Java?

    Answer: `Semaphore` is a concurrency utility that controls access to a particular resource by multiple threads. It maintains a set of permits, and a thread must acquire a permit before accessing the resource. It helps manage resource pooling and rate limiting.

Set 2

  1. What is `ExecutorService`'s `shutdown()` method used for?

    Answer: The `shutdown()` method initiates an orderly shutdown of the `ExecutorService`. It prevents new tasks from being submitted but allows previously submitted tasks to complete.

  2. What is the difference between `ThreadPoolExecutor` and `ForkJoinPool`?

    Answer:

    • `ThreadPoolExecutor`: Optimized for executing tasks that don't need to be recursively split into smaller tasks.
    • `ForkJoinPool`: Designed for recursive tasks that can be divided into smaller tasks, enabling parallel processing for divide-and-conquer algorithms.

  3. What does the `CyclicBarrier` class do in Java?

    Answer: `CyclicBarrier` is a synchronization aid that allows a set of threads to all wait for each other to reach a common barrier point. Once all threads reach the barrier, they can all proceed simultaneously.

    import java.util.concurrent.*;

public class Main {
    public static void main(String[] args) throws InterruptedException {
        CyclicBarrier barrier = new CyclicBarrier(3, () -> System.out.println("All threads are ready"));

        Runnable task = () -> {
            try {
                System.out.println(Thread.currentThread().getName() + " is waiting");
                barrier.await();
                System.out.println(Thread.currentThread().getName() + " passed the barrier");
            } catch (InterruptedException | BrokenBarrierException e) {
                Thread.currentThread().interrupt();
            }
        };

        new Thread(task).start();
        new Thread(task).start();
        new Thread(task).start();
    }
}
  4. What is the `Future.get()` method used for?

    Answer: The `Future.get()` method is used to retrieve the result of a task that has been submitted to an `ExecutorService`. It blocks the calling thread until the task completes and returns the result.

  5. What is the purpose of the `ThreadLocal` class?

    Answer: `ThreadLocal` provides thread-local variables. Each thread accessing the variable has its own independent copy, which is not shared with other threads, making it useful for maintaining thread-specific state.

    public class Main {
    private static final ThreadLocal threadLocalValue = ThreadLocal.withInitial(() -> 0);
    
    public static void main(String[] args) {
        Runnable task = () -> {
            threadLocalValue.set(threadLocalValue.get() + 1);
            System.out.println(Thread.currentThread().getName() + ": " + threadLocalValue.get());
        };

        new Thread(task).start();
        new Thread(task).start();
    }
}
  6. What is a "deadlock" in multithreading?

    Answer: A deadlock occurs when two or more threads are blocked forever, each waiting on the other to release resources. This results in a situation where no thread can proceed.

  7. What is the difference between `synchronized` blocks and `ReentrantLock`?

    Answer:

    • `synchronized`: Implicit locking, limited to methods or blocks of code, and cannot be interrupted.
    • `ReentrantLock`: Explicit locking with more control, including features like try-lock, timed-lock, and interruptible lock acquisition.

  8. How does the `ForkJoinPool` work in Java?

    Answer: `ForkJoinPool` works by recursively breaking down tasks into smaller sub-tasks (forking), processing them in parallel, and then combining the results (joining). It is designed for divide-and-conquer algorithms that require recursive task splitting.

  9. What is the `join()` method in the context of threads?

    Answer: The `join()` method allows one thread to wait for the completion of another. It blocks the calling thread until the thread whose `join()` method is invoked has finished its execution.

    public class Main {
    public static void main(String[] args) throws InterruptedException {
        Thread thread = new Thread(() -> System.out.println("Thread is running"));
        thread.start();
        thread.join(); // Main thread waits for the above thread to finish
        System.out.println("Main thread continues");
    }
}
  10. What is the `Lock.tryLock()` method used for?

    Answer: The `tryLock()` method attempts to acquire a lock without blocking the thread. If the lock is not available, it immediately returns `false`, allowing the thread to continue with other tasks. It can be used with a timeout option as well.

  1. What is a `CountDownLatch` in Java?

    Answer: `CountDownLatch` is a synchronization aid that allows one or more threads to wait until a set of operations in other threads completes. It is initialized with a count, and each thread calls `countDown()` when it completes an operation. The main thread can call `await()` to wait until the count reaches zero.

    import java.util.concurrent.*;

public class Main {
    public static void main(String[] args) throws InterruptedException {
        CountDownLatch latch = new CountDownLatch(3);

        Runnable task = () -> {
            try {
                System.out.println(Thread.currentThread().getName() + " is working");
                Thread.sleep(1000);
                latch.countDown();
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        };

        new Thread(task).start();
        new Thread(task).start();
        new Thread(task).start();

        latch.await(); // Main thread waits for latch to reach zero
        System.out.println("All threads are done");
    }
}
  2. How does `ExecutorService.submit()` differ from `ExecutorService.execute()`?

    Answer: `submit()` returns a `Future` object which can be used to retrieve the result or exception of a task, whereas `execute()` is a void method that does not return a result or handle exceptions. `submit()` is useful when you need to get the result of the task.

  3. What is the `Semaphore` class used for in Java concurrency?

    Answer: `Semaphore` is a counting semaphore that controls access to a shared resource by a set of threads. It maintains a set of permits, and threads acquire permits before proceeding with execution. If no permits are available, threads are blocked until permits are released.

  4. What is the purpose of `ReentrantReadWriteLock` in Java?

    Answer: `ReentrantReadWriteLock` is a lock that allows multiple threads to read a resource concurrently but only one thread to write to it. It provides separate read and write locks to improve performance by allowing multiple readers.

  5. What is the difference between `synchronized` keyword and `ReentrantLock`?

    Answer:

    • `synchronized`: Implicit lock, simpler to use but lacks features such as try-lock and interruptible lock acquisition.
    • `ReentrantLock`: Explicit lock, provides additional features such as try-lock, timed lock, and interruptible lock acquisition.

  6. What is the difference between `ForkJoinTask` and `Future`?

    Answer: `ForkJoinTask` is a special subclass of `Future` designed for tasks that can be recursively split into smaller tasks (i.e., divide-and-conquer tasks). `Future` is a general-purpose interface used to represent the result of any asynchronous computation.

  7. What is the purpose of the `Object.wait()` method in multithreading?

    Answer: `wait()` is used to pause the execution of the current thread until another thread sends a notification via `notify()` or `notifyAll()`. It is typically used in synchronized blocks to facilitate inter-thread communication.

    public class Main {
    private static final Object lock = new Object();

    public static void main(String[] args) throws InterruptedException {
        Thread t1 = new Thread(() -> {
            synchronized (lock) {
                try {
                    lock.wait();
                    System.out.println("Thread resumed after wait");
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                }
            }
        });

        t1.start();
        Thread.sleep(1000);
        synchronized (lock) {
            lock.notify();
        }
    }
}
  8. How does `ExecutorService.invokeAll()` differ from `ExecutorService.invokeAny()`?

    Answer:

    • `invokeAll()`: Executes all tasks and returns a list of `Future` objects, blocking until all tasks are completed.
    • `invokeAny()`: Executes the tasks and returns the result of the first completed task, ignoring other tasks once one completes.

  9. What is the `ThreadPoolExecutor` class used for?

    Answer: `ThreadPoolExecutor` is an implementation of the `ExecutorService` interface that provides a pool of worker threads for executing tasks. It allows you to manage the number of threads dynamically based on the load, improving performance by reusing threads.

  10. What is the `Lock` interface used for?

    Answer: The `Lock` interface provides a more flexible locking mechanism than `synchronized` blocks, allowing features such as try-lock, timed lock, and interruptible lock acquisition. It is part of the `java.util.concurrent.locks` package.

Set 4

  1. What is `ExecutorService` in Java?

    Answer: `ExecutorService` is an interface in the `java.util.concurrent` package that represents an asynchronous execution mechanism, allowing you to submit tasks for execution without manually managing threads. Common implementations include `ThreadPoolExecutor` and `ScheduledThreadPoolExecutor`.

  2. How does `CyclicBarrier` work in Java?

    Answer: `CyclicBarrier` is a synchronization aid that allows a set of threads to all wait for each other to reach a common barrier point. Once all threads reach the barrier, they can continue execution. It can be reused once the barrier is tripped.

    import java.util.concurrent.*;

public class Main {
    public static void main(String[] args) throws InterruptedException {
        CyclicBarrier barrier = new CyclicBarrier(3, () -> System.out.println("Barrier reached"));

        Runnable task = () -> {
            try {
                System.out.println(Thread.currentThread().getName() + " is waiting");
                barrier.await();
                System.out.println(Thread.currentThread().getName() + " has passed the barrier");
            } catch (InterruptedException | BrokenBarrierException e) {
                Thread.currentThread().interrupt();
            }
        };

        new Thread(task).start();
        new Thread(task).start();
        new Thread(task).start();
    }
}
  3. What is the `ForkJoinPool` in Java and when is it used?

    Answer: `ForkJoinPool` is a special kind of `ExecutorService` designed to efficiently process tasks that can be recursively split into smaller tasks. It uses a work-stealing algorithm to minimize the time spent on idle threads, making it ideal for divide-and-conquer tasks.

  4. What is the purpose of `volatile` keyword in multithreading?

    Answer: The `volatile` keyword in Java ensures that a variable is directly read from and written to the main memory, rather than from the local thread cache. This helps ensure visibility of changes across all threads, although it does not guarantee atomicity.

  5. What is the difference between `Thread.sleep()` and `Object.wait()`?

    Answer:

    • `Thread.sleep()`: Pauses the execution of the current thread for a specified time, without releasing any locks.
    • `Object.wait()`: Pauses the execution of the current thread until another thread sends a notification via `notify()` or `notifyAll()`. It must be called inside a synchronized block.

  6. What are the main methods of `ReentrantLock` in Java?

    Answer: The main methods of `ReentrantLock` include:

    • `lock()`: Acquires the lock.
    • `unlock()`: Releases the lock.
    • `tryLock()`: Attempts to acquire the lock without blocking.
    • `lockInterruptibly()`: Acquires the lock, but can be interrupted.

  7. What does `ExecutorService.invokeAll()` return?

    Answer: `invokeAll()` returns a list of `Future` objects, each representing the result of one task. It waits for all tasks to complete before returning, and guarantees that all tasks are executed.

  8. What is `CompletableFuture` in Java?

    Answer: `CompletableFuture` is a class in Java that represents a future result of an asynchronous computation. It provides methods to explicitly complete the future, combine multiple futures, and handle errors in a more flexible way than `Future`.

    import java.util.concurrent.*;

public class Main {
    public static void main(String[] args) {
        CompletableFuture.supplyAsync(() -> {
            return "Hello from CompletableFuture!";
        }).thenAccept(result -> {
            System.out.println(result);
        });
    }
}
  9. How do `notify()` and `notifyAll()` differ in multithreading?

    Answer:

    • `notify()`: Wakes up a single thread that is waiting on the object's monitor.
    • `notifyAll()`: Wakes up all threads that are waiting on the object's monitor.

  10. What is the significance of `ThreadLocal` in Java?

    Answer: `ThreadLocal` provides thread-local variables, meaning each thread has its own independent copy of the variable. This can be useful for storing data that is unique to the thread, such as user sessions or database connections.

      import java.util.concurrent.*;

public class ForkJoinExample extends RecursiveTask {
    private int n;

    ForkJoinExample(int n) {
        this.n = n;
    }

    @Override
    protected Integer compute() {
        if (n <= 1) {
            return n;
        }
        ForkJoinExample subTask1 = new ForkJoinExample(n - 1);
        subTask1.fork();
        return n + subTask1.join();
    }

    public static void main(String[] args) {
        ForkJoinPool forkJoinPool = new ForkJoinPool();
        ForkJoinExample task = new ForkJoinExample(10);
        System.out.println("Result: " + forkJoinPool.invoke(task));
    }
}

    4. What does `ReentrantLock.tryLock()` do?

      Answer: `tryLock()` attempts to acquire the lock without blocking. If the lock is available, it acquires the lock and returns `true`; otherwise, it returns `false` immediately.

    5. How does the `Callable` interface differ from `Runnable`?

      Answer: `Callable` is similar to `Runnable`, but it allows the task to return a result and throw exceptions. `Runnable` does not return any result and cannot throw checked exceptions.

    6. What are the advantages of using `ExecutorService` over manually creating threads?

      Answer: `ExecutorService` simplifies thread management by reusing a pool of threads, avoiding the overhead of creating new threads repeatedly. It also handles the execution and termination of tasks, error handling, and can be shut down gracefully.

    7. What is the difference between `synchronized` blocks and `Lock` objects?

      Answer: `synchronized` is a language feature that provides an implicit lock for a method or block of code, whereas `Lock` is a more flexible API that provides additional methods for acquiring, releasing, and checking locks, such as timed locking and interruptible lock waits.

    8. What is a `CountDownLatch` used for in Java concurrency?

      Answer: `CountDownLatch` is used to block a thread until a specified number of events have occurred. It is often used for waiting for multiple threads to finish their tasks before proceeding.

    9. Explain how `ReentrantLock` works in Java.

      Answer: `ReentrantLock` is a lock that allows a thread to acquire the same lock multiple times without causing deadlock. It is reentrant because the thread that holds the lock can enter into the synchronized block or method again without being blocked.

    10. What is `Thread.join()` used for?

      Answer: `Thread.join()` is used to make the calling thread wait until the thread on which `join()` was called has finished its execution. It ensures that one thread waits for the completion of another before continuing.

     

    Concurrency & Multithreading - Questions & Answers

    Set 10

    1. What is the difference between `ExecutorService.submit()` and `ExecutorService.execute()`?

      Answer: `submit()` returns a `Future` object that can be used to retrieve the result or handle exceptions, while `execute()` does not return any result and is used for tasks that do not require a result or exception handling.

    2. What is the purpose of `ForkJoinTask`?

      Answer: `ForkJoinTask` is the base class for tasks that can be executed in a `ForkJoinPool`. It allows tasks to be recursively split into smaller sub-tasks and joined together to compute a result.

    3. What does `synchronized` in Java prevent?

      Answer: `synchronized` ensures that only one thread can access a critical section of code at a time, preventing race conditions and ensuring thread safety.

    4. How does `Semaphore` work in Java concurrency?

      Answer: `Semaphore` controls access to a shared resource by maintaining a set of permits. Threads acquire a permit before proceeding and release it after finishing their task. It is often used to limit the number of threads that can access a resource simultaneously.

    5. What is the role of `ExecutorService.shutdown()` method?

      Answer: The `shutdown()` method in `ExecutorService` initiates an orderly shutdown of the executor, where previously submitted tasks are executed, but no new tasks will be accepted.

    6. What is the purpose of `CyclicBarrier` in Java concurrency?

      Answer: `CyclicBarrier` allows a set of threads to wait for each other to reach a common barrier point. Once all threads reach the barrier, they are released to continue execution. It is often used in scenarios where threads need to synchronize their progress.

    7. What is `AtomicInteger` in Java and how does it work?

      Answer: `AtomicInteger` is a class that provides atomic operations on integers. It ensures that updates to the integer value are done atomically, avoiding race conditions without using synchronization.

    8. What is `Lock`'s `lockInterruptibly()` method used for?

      Answer: The `lockInterruptibly()` method allows a thread to acquire a lock unless it is interrupted. If the thread is interrupted while trying to acquire the lock, it throws an `InterruptedException`, which gives more control over thread interruption handling compared to `lock()`.

    9. What is the `ThreadPoolExecutor` class used for?

      Answer: `ThreadPoolExecutor` is a flexible and powerful implementation of the `ExecutorService` interface. It manages a pool of worker threads to execute tasks concurrently, providing features like core and maximum pool size, task queue, and thread factory.

    10. What is the difference between `wait()` and `sleep()` in Java?

      Answer: `wait()` is used for inter-thread communication and requires the thread to hold a lock on the object, releasing it when the thread is put into a waiting state. `sleep()` simply pauses the thread for a specified amount of time without releasing any locks.
