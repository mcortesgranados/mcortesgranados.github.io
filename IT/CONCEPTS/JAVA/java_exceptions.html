Exception Handling in Java - Questions & Answers

1. What is the difference between checked and unchecked exceptions in Java?

In Java, exceptions are divided into two categories:

  • Checked Exceptions: These are exceptions that are checked at compile-time. The compiler requires that these exceptions be either handled using a try-catch block or declared in the method signature using the `throws` keyword. Examples include `IOException` and `SQLException`.
  • Unchecked Exceptions: These are exceptions that occur at runtime and are not checked by the compiler. They are subclasses of `RuntimeException` and can occur due to programming errors. Examples include `NullPointerException`, `ArrayIndexOutOfBoundsException`, and `ArithmeticException`.

2. What is a try-catch-finally block, and how would you use it?

A try-catch-finally block is used to handle exceptions in Java:

  • try: The code that might throw an exception is placed inside the try block.
  • catch: If an exception occurs in the try block, it is caught by the corresponding catch block, where the exception can be handled.
  • finally: The code inside the finally block always runs, regardless of whether an exception was thrown or not. It is typically used for cleanup actions (e.g., closing resources).

Example:

    try {
        // Code that may throw an exception
        int result = 10 / 0; // ArithmeticException
    } catch (ArithmeticException e) {
        System.out.println("Error: Division by zero.");
    } finally {
        System.out.println("This block always runs.");
    }

3. What is the difference between throw and throws?

throw and throws are both used for exception handling, but they serve different purposes:

  • throw: The throw keyword is used to explicitly throw an exception from within a method or block of code. It is followed by an instance of an exception class.
  • throws: The throws keyword is used in a method declaration to indicate that the method may throw one or more exceptions. It does not throw the exception itself but rather signals that the caller of the method must handle it.

Example:

    // Using throw to throw an exception
    public void myMethod() {
        throw new IllegalArgumentException("Invalid argument");
    }

    // Using throws to declare an exception
    public void myMethod() throws IOException {
        // Code that may throw an IOException
    }

4. What happens if an exception is not caught inside a try-catch block?

If an exception is not caught within a try-catch block, it will propagate up the call stack. If the exception is not handled at any point in the call stack, it will eventually cause the program to terminate, and the exception will be printed along with its stack trace. Uncaught exceptions in the main method will result in the termination of the program.

Exception Handling in Java - Additional Questions & Answers

6. What is the difference between `throw` and `throws` in terms of syntax?

The throw keyword is used to explicitly throw an exception, followed by the instance of the exception. The throws keyword is used in method declarations to specify that a method might throw an exception.

7. What is a `NullPointerException`, and how can it be avoided?

A NullPointerException occurs when you try to access a method or field of an object that is null. To avoid it, ensure that objects are initialized before use, or check for null before performing operations.

8. What is the purpose of the `finally` block?

The finally block is used to execute code that must run after a try-catch block, whether an exception was thrown or not. It is typically used for cleanup operations such as closing files or releasing resources.

9. Can you catch multiple exceptions in one catch block?

Yes, since Java 7, you can catch multiple exceptions in a single catch block by separating them with a vertical bar (|). This is called multi-catch.

    try {
        // Code that may throw exceptions
    } catch (IOException | SQLException e) {
        System.out.println("Exception occurred: " + e.getMessage());
    }

10. What is the `Exception` class in Java?

The Exception class is the superclass of all exceptions in Java, except for errors. It is used as a base class for creating custom exceptions and handling exceptions that may occur during program execution.

11. What is an `ArithmeticException`?

An ArithmeticException is thrown when an exceptional arithmetic condition occurs, such as division by zero.

12. How do you handle `ArrayIndexOutOfBoundsException`?

An ArrayIndexOutOfBoundsException occurs when you try to access an index outside the bounds of an array. To avoid this exception, ensure that the index is within valid bounds before accessing array elements.

13. What is a `ClassNotFoundException`?

A ClassNotFoundException is thrown when an application attempts to load a class by its name, but the class cannot be found on the classpath.

14. Can you have a `finally` block without a `catch` block?

Yes, you can have a `finally` block without a `catch` block. This is useful when you want to ensure some cleanup code runs, even if no exception occurs.

    try {
        // Code that might throw an exception
    } finally {
        System.out.println("This block always runs.");
    }

15. What is the difference between `RuntimeException` and `Exception`?

RuntimeException is a subclass of Exception and represents exceptions that are thrown during the execution of a program due to programming errors. Exception is a more general class for checked exceptions.

16. How would you handle multiple exceptions using multiple catch blocks?

In Java, you can use multiple catch blocks to handle different types of exceptions separately. Each catch block should handle one specific type of exception.

    try {
        // Code that may throw different exceptions
    } catch (IOException e) {
        System.out.println("IOException caught");
    } catch (SQLException e) {
        System.out.println("SQLException caught");
    }

17. What is the purpose of the `try-with-resources` statement?

The try-with-resources statement ensures that each resource is closed at the end of the statement. It is typically used with classes that implement the `AutoCloseable` interface, such as `FileInputStream` or `BufferedReader`.

18. What is the `StackOverflowError`?

A StackOverflowError occurs when the call stack for a program exceeds its limit, usually due to excessive recursion. This error cannot be caught and is a subclass of `Error`.

19. Can exceptions be thrown from a constructor?

Yes, exceptions can be thrown from a constructor if an error occurs while creating an object. If an exception is thrown in the constructor, the object is not created.

20. How can you create a custom exception that extends a checked exception?

To create a custom checked exception, you need to extend the `Exception` class or one of its subclasses. It must be declared using the throws keyword in methods that throw it.

    public class MyCheckedException extends Exception {
        public MyCheckedException(String message) {
            super(message);
        }
    }

21. How can you re-throw an exception in Java?

You can re-throw an exception by using the throw keyword within a catch block. This allows the exception to propagate further up the call stack.

    try {
        // Code that may throw an exception
    } catch (IOException e) {
        System.out.println("Exception caught, rethrowing...");
        throw e;
    }

22. What is an `IndexOutOfBoundsException`?

An IndexOutOfBoundsException is thrown when an index used to access an element in a list or array is outside the valid range (e.g., negative index or index larger than the array length).

23. What is the difference between `Error` and `Exception`?

Both `Error` and `Exception` are throwable classes in Java, but errors are typically not recoverable (e.g., `OutOfMemoryError`), whereas exceptions represent conditions that can be handled by the program.

24. How would you handle `IOException` when working with files?

To handle an IOException, you can use a `try-catch` block around file-related operations and properly close file streams in the `finally` block or by using `try-with-resources`.

25. What is `IllegalArgumentException`?

An IllegalArgumentException is thrown to indicate that a method has been passed an illegal or inappropriate argument.

26. Can you catch `Throwable` in Java?

Yes, but it is not recommended to catch `Throwable` in general, as it includes errors (e.g., `OutOfMemoryError`) that are beyond the control of the program and typically should not be caught.

27. How can you handle multiple exceptions in a single catch block?

Since Java 7, you can use the multi-catch syntax to handle multiple exceptions in a single catch block, which makes the code cleaner and more concise.

    try {
        // Code that may throw multiple exceptions
    } catch (IOException | SQLException e) {
        System.out.println("Exception occurred: " + e.getMessage());
    }

28. What is a `FileNotFoundException`?

A FileNotFoundException occurs when an application tries to open a file that does not exist or is inaccessible.

29. How do you handle exceptions in multi-threaded applications?

In multi-threaded applications, exceptions can be handled by wrapping thread execution in a try-catch block within the thread's run method, or by using a custom exception handler for each thread.

30. Can exceptions be caught inside a static block?

Yes, exceptions can be caught inside a static block, and the same rules apply as with instance initialization blocks. You can use try-catch blocks to handle any exceptions that occur within the static block.
