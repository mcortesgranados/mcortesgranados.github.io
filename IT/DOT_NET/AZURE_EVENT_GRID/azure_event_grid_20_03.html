Azure Event Grid Q&A - Part 03

1. What is the pricing model for Azure Event Grid?

Azure Event Grid pricing is based on the number of operations, including event publishing and management operations, with a pay-as-you-go model.

2. How can you filter events in Azure Event Grid?

Event filtering can be done based on event types, subject begins/ends with, and custom attributes specified in the event schema.

3. Can Azure Event Grid send events to multiple endpoints simultaneously?

No, Azure Event Grid sends events to one endpoint per event subscription. However, you can create multiple subscriptions to the same source to send to different endpoints.

4. What is the role of the Event Grid schema?

The Event Grid schema defines the structure of events published to Event Grid, including standard properties like event type, subject, and data.

5. How do you handle failed event deliveries in Azure Event Grid?

Failed deliveries can be handled using retries and by configuring dead-letter destinations where undeliverable events can be sent for further analysis.

6. What kind of events can be published to Azure Event Grid?

Events can be published from Azure services like Blob Storage, Resource Manager, and custom applications, covering a wide range of scenarios.

7. Can you use Azure Event Grid with serverless architectures?

Yes, Azure Event Grid works seamlessly with serverless architectures, integrating well with Azure Functions and Logic Apps for event-driven processing.

8. What happens to events if the endpoint is unavailable?

If the endpoint is unavailable, Azure Event Grid will retry delivery for a specified period, and if still unsuccessful, the event may be sent to a dead-letter destination.

9. Can Azure Event Grid support custom events?

Yes, Azure Event Grid supports custom events, allowing applications to publish and subscribe to their own defined events.

10. What is a custom topic in Azure Event Grid?

A custom topic is a user-defined topic where you can publish events, separate from the system topics that Azure services use.

11. How do you create an event subscription in Azure Event Grid?

An event subscription can be created using the Azure portal, Azure CLI, or REST API by specifying the event source and endpoint for delivery.

12. Can Azure Event Grid deliver events to on-premises applications?

Yes, events can be delivered to on-premises applications by using a hybrid connection or a publicly accessible endpoint with appropriate authentication.

13. How can Azure Event Grid be monitored?

Monitoring can be done using Azure Monitor, enabling metrics, logs, and alerts to track event delivery, failures, and performance.

14. Can you configure a retry policy for Azure Event Grid?

Yes, Azure Event Grid has a built-in retry policy that automatically retries delivery for up to 24 hours before considering the event undeliverable.

15. What is the difference between an event subscription and an event handler?

An event subscription defines how events are routed to a destination, while an event handler is the application or service that processes the received events.

16. How do you delete a topic in Azure Event Grid?

A topic can be deleted through the Azure portal, Azure CLI, or REST API by specifying the topic's name and the resource group it belongs to.

17. What authentication methods can be used with Azure Event Grid?

Authentication methods include Azure Active Directory, managed identities, and shared access signatures (SAS) for securing event delivery.

18. Can Azure Event Grid be used in multi-cloud environments?

While Azure Event Grid is primarily an Azure service, it can integrate with applications in other clouds via webhooks and APIs, enabling multi-cloud scenarios.

19. How do you validate event delivery in Azure Event Grid?

Validation can be done by monitoring delivery status through Azure Monitor or by implementing logging in the endpoint to confirm receipt of events.

20. What is the role of the Azure SDK in using Event Grid?

The Azure SDK provides libraries and tools to simplify interaction with Azure Event Grid, making it easier to publish events, create subscriptions, and manage resources programmatically.
