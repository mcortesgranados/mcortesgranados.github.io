C# Structs

Structs in C# are value types that provide a lightweight alternative to classes for small, simple objects. They are ideal for representing small data structures that have value semantics rather than reference semantics.

1. Purpose

Structs are used to:

  • Encapsulate Small Data: Represent small, related data items together.
  • Improve Performance: Avoid the overhead of heap allocation and garbage collection.
  • Use Value Semantics: Provide value-based equality and copying.

2. Defining a Struct

To define a struct, use the struct keyword. Here’s an example of a simple struct definition:


using System;

public struct Point
{
    public int X { get; set; }
    public int Y { get; set; }

    // Constructor
    public Point(int x, int y)
    {
        X = x;
        Y = y;
    }

    // Method
    public void Display()
    {
        Console.WriteLine($"Point: ({X}, {Y})");
    }
}

3. Creating and Using Structs

To use a struct, create an instance and access its members. Here’s an example:


using System;

class Program
{
    static void Main()
    {
        // Create an instance of the struct
        Point point = new Point(10, 20);

        // Access and use the struct's members
        point.Display();

        // Modify the struct's properties
        point.X = 30;
        point.Y = 40;
        point.Display();
    }
}

4. Key Points

  • Value Type: Structs are value types and are stored on the stack.
  • No Inheritance: Structs do not support inheritance, but they can implement interfaces.
  • Default Constructor: Structs do not have a parameterless default constructor; you must initialize all fields.
  • Immutability: Structs are often used to represent immutable data (e.g., Point, Color).
  • Performance: Structs can be more efficient than classes for small, frequently created objects due to their stack allocation.
