C# Using Statements

The using statement in C# is used for two primary purposes:

  • Namespace Import: To include namespaces in the current file so that you can use the types defined in those namespaces without needing to fully qualify their names.
  • Resource Management: To ensure that resources are properly disposed of once they are no longer needed, which is particularly useful for handling unmanaged resources.

1. Using Statements for Namespace Import

The using directive allows you to use types from a namespace without fully qualifying them. Here’s an example:


using System;

namespace MyApplication
{
    class Program
    {
        static void Main()
        {
            Console.WriteLine("Hello, World!");
        }
    }
}

In this example, the using System; directive imports the System namespace, so you can use Console without needing to write System.Console.

2. Using Statements for Resource Management

The using statement can also be used to automatically dispose of resources when they are no longer needed. This is typically used with objects that implement the IDisposable interface. Here’s an example:


using System;
using System.IO;

namespace MyApplication
{
    class Program
    {
        static void Main()
        {
            using (StreamWriter writer = new StreamWriter("example.txt"))
            {
                writer.WriteLine("Hello, World!");
            }  // StreamWriter is disposed of here
        }
    }
}

In this example, the using statement ensures that the StreamWriter is disposed of properly after its usage, even if an exception occurs.

3. Nested Using Statements

You can nest using statements to manage multiple resources:


using (StreamReader reader = new StreamReader("input.txt"))
{
    using (StreamWriter writer = new StreamWriter("output.txt"))
    {
        string line;
        while ((line = reader.ReadLine()) != null)
        {
            writer.WriteLine(line);
        }
    }
}

In this example, both the StreamReader and StreamWriter are disposed of correctly at the end of their usage.

4. Key Points

  • Namespace Import: Simplifies code by allowing types to be used without full qualification.
  • Resource Management: Automatically disposes of resources that implement IDisposable to prevent resource leaks.
  • Nested Using Statements: Can be used to manage multiple resources, ensuring each is disposed of properly.
