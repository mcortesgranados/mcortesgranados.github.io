C# Virtual Methods

In C#, a virtual method is a method that can be overridden in a derived class, allowing for dynamic method binding. This enables polymorphism, where the derived class can provide a specific implementation of the method, while the base class provides a default implementation.

Key Points of Virtual Methods:

  • To declare a virtual method, use the virtual keyword in the base class.
  • To override a virtual method, the derived class uses the override keyword.
  • If the derived class does not override the method, the base class implementation is used.
  • Virtual methods enable runtime polymorphism, allowing derived classes to define their own behavior.

Example of a Virtual Method in C#

In this example, we have a base class Animal with a virtual method MakeSound(). The Dog and Cat classes override this method to provide their own implementations.


// Base class
public class Animal
{
    // Virtual method
    public virtual void MakeSound()
    {
        Console.WriteLine("Animal makes a sound.");
    }
}

// Derived class: Dog
public class Dog : Animal
{
    // Override virtual method
    public override void MakeSound()
    {
        Console.WriteLine("Dog barks.");
    }
}

// Derived class: Cat
public class Cat : Animal
{
    // Override virtual method
    public override void MakeSound()
    {
        Console.WriteLine("Cat meows.");
    }
}

// Main program
public class Program
{
    public static void Main()
    {
        Animal myAnimal = new Animal();
        Animal myDog = new Dog();
        Animal myCat = new Cat();

        myAnimal.MakeSound(); // Output: Animal makes a sound.
        myDog.MakeSound();    // Output: Dog barks.
        myCat.MakeSound();    // Output: Cat meows.
    }
}

Explanation:

In the above code:

  • Animal is the base class with a virtual method MakeSound().
  • The Dog and Cat classes inherit from Animal and override the MakeSound() method with their specific behavior.
  • At runtime, when the MakeSound() method is called on an Animal reference, the derived class's method (if overridden) is executed.
  • This demonstrates polymorphism where the method behavior changes based on the object's type at runtime.

Conclusion

Virtual methods in C# allow derived classes to provide their own implementations of a method. They enable polymorphism and dynamic method binding, which makes the code more flexible and extendable. When a method is declared as virtual, it can be overridden by any class that inherits from the base class, allowing each derived class to have its own specific behavior.
