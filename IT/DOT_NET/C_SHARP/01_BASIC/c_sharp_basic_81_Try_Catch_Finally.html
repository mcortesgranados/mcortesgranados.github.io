C# Try, Catch, Finally

In C#, exception handling is managed using the try, catch, and finally blocks. These blocks help manage runtime errors and ensure that resources are properly released.

1. Try Block

The try block contains the code that might throw an exception. If an exception occurs, the control is transferred to the catch block.


try
{
    // Code that may throw an exception
    int result = 10 / 0; // This will throw a DivideByZeroException
}

2. Catch Block

The catch block handles the exception thrown by the try block. You can have multiple catch blocks to handle different types of exceptions.


try
{
    // Code that may throw an exception
    int result = 10 / 0; // This will throw a DivideByZeroException
}
catch (DivideByZeroException ex)
{
    // Handle DivideByZeroException
    Console.WriteLine("Cannot divide by zero.");
}
catch (Exception ex)
{
    // Handle any other exceptions
    Console.WriteLine($"An error occurred: {ex.Message}");
}

3. Finally Block

The finally block contains code that will always execute regardless of whether an exception was thrown or not. It's typically used for cleanup activities, such as closing files or releasing resources.


try
{
    // Code that may throw an exception
    int result = 10 / 0; // This will throw a DivideByZeroException
}
catch (DivideByZeroException ex)
{
    // Handle DivideByZeroException
    Console.WriteLine("Cannot divide by zero.");
}
finally
{
    // Code that will always run
    Console.WriteLine("This will always execute.");
}

4. Key Points

  • Try Block: Encloses code that may throw an exception.
  • Catch Block: Handles specific or general exceptions.
  • Finally Block: Executes code after try and catch blocks, regardless of whether an exception was thrown.

5. Conclusion

The try, catch, and finally blocks in C# provide a structured approach to handling exceptions and ensuring that necessary cleanup operations are performed. Proper use of these blocks helps in creating robust and error-resilient applications.
