C# Indexers

Indexers in C# allow objects to be indexed in a way similar to arrays. They enable instances of a class or struct to be indexed just like arrays, providing a way to access elements in an object using an indexer syntax. This feature is particularly useful for classes that represent collections of data.

1. Purpose

Indexers are used to:

  • Enable Array-Like Access: Allow objects to be accessed using index notation.
  • Encapsulate Data: Provide controlled access to the data stored within a class or struct.
  • Implement Collection Classes: Simplify the interaction with classes that manage collections of items.

2. Declaring an Indexer

To declare an indexer, use the this keyword followed by a parameter list. You can define the getter and setter to control access to the data.


    public class StringCollection
    {
        private string[] _strings = new string[10];

        public string this[int index]
        {
            get
            {
                if (index >= 0 && index < _strings.Length)
                {
                    return _strings[index];
                }
                else
                {
                    throw new IndexOutOfRangeException("Index out of range.");
                }
            }
            set
            {
                if (index >= 0 && index < _strings.Length)
                {
                    _strings[index] = value;
                }
                else
                {
                    throw new IndexOutOfRangeException("Index out of range.");
                }
            }
        }
    }

3. Using an Indexer

Once declared, you can use an indexer to access elements of the collection using index notation, similar to how you would with an array.


    class Program
    {
        static void Main()
        {
            StringCollection collection = new StringCollection();

            collection[0] = "Hello";
            collection[1] = "World";

            Console.WriteLine(collection[0]); // Output: Hello
            Console.WriteLine(collection[1]); // Output: World
        }
    }

4. Indexer with Multiple Parameters

Indexers can also take multiple parameters to support more complex indexing scenarios.


    public class MultiDimensionalCollection
    {
        private int[,] _matrix = new int[3, 3];

        public int this[int row, int col]
        {
            get
            {
                if (row >= 0 && row < _matrix.GetLength(0) && col >= 0 && col < _matrix.GetLength(1))
                {
                    return _matrix[row, col];
                }
                else
                {
                    throw new IndexOutOfRangeException("Index out of range.");
                }
            }
            set
            {
                if (row >= 0 && row < _matrix.GetLength(0) && col >= 0 && col < _matrix.GetLength(1))
                {
                    _matrix[row, col] = value;
                }
                else
                {
                    throw new IndexOutOfRangeException("Index out of range.");
                }
            }
        }
    }

5. Key Points

  • Indexers allow objects to be indexed using a custom indexer syntax.
  • Indexers provide a way to access elements in a collection-like class or struct.
  • Getter and setter methods can be defined to control access to the indexed elements.
  • Multiple parameters can be used to support multidimensional indexing.
