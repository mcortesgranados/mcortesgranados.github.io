C# Collections

Collections in C# are data structures that hold multiple values. They are essential for managing and manipulating groups of objects. C# provides various types of collections, each designed for different scenarios.

1. Arrays

Arrays are a basic collection type that holds a fixed-size sequence of elements of the same type.


    using System;

    class Program
    {
        static void Main()
        {
            // Declare and initialize an array
            int[] numbers = { 1, 2, 3, 4, 5 };

            // Access array elements
            foreach (int number in numbers)
            {
                Console.WriteLine(number);
            }
        }
    }

2. Lists

The List<T> class in the System.Collections.Generic namespace provides a dynamic array that can grow as needed.


    using System;
    using System.Collections.Generic;

    class Program
    {
        static void Main()
        {
            // Create and initialize a list
            List<string> names = new List<string> { "Alice", "Bob", "Charlie" };

            // Add an item to the list
            names.Add("David");

            // Access list elements
            foreach (string name in names)
            {
                Console.WriteLine(name);
            }
        }
    }

3. Dictionaries

The Dictionary<TKey, TValue> class in the System.Collections.Generic namespace stores key-value pairs.


    using System;
    using System.Collections.Generic;

    class Program
    {
        static void Main()
        {
            // Create and initialize a dictionary
            Dictionary<string, int> ageDictionary = new Dictionary<string, int>()
            {
                { "Alice", 30 },
                { "Bob", 25 }
            };

            // Add an item to the dictionary
            ageDictionary["Charlie"] = 35;

            // Access dictionary elements
            foreach (var kvp in ageDictionary)
            {
                Console.WriteLine($"{kvp.Key}: {kvp.Value}");
            }
        }
    }

4. Queues

The Queue<T> class in the System.Collections.Generic namespace represents a first-in, first-out (FIFO) collection.


    using System;
    using System.Collections.Generic;

    class Program
    {
        static void Main()
        {
            // Create and initialize a queue
            Queue<string> queue = new Queue<string>();
            queue.Enqueue("First");
            queue.Enqueue("Second");
            queue.Enqueue("Third");

            // Dequeue and display elements
            while (queue.Count > 0)
            {
                Console.WriteLine(queue.Dequeue());
            }
        }
    }

5. Stacks

The Stack<T> class in the System.Collections.Generic namespace represents a last-in, first-out (LIFO) collection.


    using System;
    using System.Collections.Generic;

    class Program
    {
        static void Main()
        {
            // Create and initialize a stack
            Stack<string> stack = new Stack<string>();
            stack.Push("First");
            stack.Push("Second");
            stack.Push("Third");

            // Pop and display elements
            while (stack.Count > 0)
            {
                Console.WriteLine(stack.Pop());
            }
        }
    }

6. Key Points

  • Arrays provide a fixed-size collection of elements.
  • List<T> is a dynamic collection that can grow as needed.
  • Dictionary<TKey, TValue> stores key-value pairs for efficient lookups.
  • Queue<T> represents a FIFO collection, ideal for tasks requiring order preservation.
  • Stack<T> represents a LIFO collection, ideal for tasks requiring last-in, first-out access.
