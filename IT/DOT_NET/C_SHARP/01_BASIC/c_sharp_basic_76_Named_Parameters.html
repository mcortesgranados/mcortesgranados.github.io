C# Named Parameters

Named parameters in C# allow you to specify the name of the parameters when calling a method, rather than relying on the order of arguments. This can make method calls more readable and flexible, especially when dealing with methods with many parameters or default values.

1. Basics of Named Parameters

With named parameters, you can pass arguments to a method by specifying the parameter names, which allows you to skip parameters or change their order.

2. Example of Named Parameters

Hereâ€™s an example showing how to use named parameters in a method call:


public class Example
{
    // Method with multiple parameters
    public void DisplayInfo(string name, int age, string city = "Unknown", string country = "Unknown")
    {
        Console.WriteLine($"Name: {name}");
        Console.WriteLine($"Age: {age}");
        Console.WriteLine($"City: {city}");
        Console.WriteLine($"Country: {country}");
    }
}

public class Program
{
    public static void Main()
    {
        Example example = new Example();

        // Using named parameters to specify values
        example.DisplayInfo(name: "Alice", age: 30, city: "New York");

        // Using named parameters to change the order and skip some parameters
        example.DisplayInfo(age: 25, name: "Bob", country: "USA");
    }
}

Output:


Name: Alice
Age: 30
City: New York
Country: Unknown

Name: Bob
Age: 25
City: Unknown
Country: USA

3. Key Points

  • Parameter Names: Named parameters are specified by their names in the method call, not by their position.
  • Flexibility: Allows changing the order of parameters and skipping optional ones with default values.
  • Readability: Improves the readability of method calls, especially when dealing with methods that have many parameters.

4. Conclusion

Named parameters in C# enhance method call clarity and flexibility by allowing you to specify parameter names directly. This feature helps in creating more maintainable and readable code, especially in complex methods.
