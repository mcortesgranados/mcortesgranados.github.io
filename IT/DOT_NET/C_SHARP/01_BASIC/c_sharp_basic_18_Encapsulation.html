C# Encapsulation

Encapsulation is one of the four fundamental principles of object-oriented programming (OOP) that involves bundling the data (fields) and methods (functions) that operate on the data into a single unit called a class. It also involves restricting direct access to some of the object's components, which can help prevent accidental interference and misuse of the data.

1. Concepts of Encapsulation

Encapsulation in C# is achieved using access modifiers to control the visibility of the class members (fields, properties, methods). The main access modifiers are:

  • private: The member is accessible only within its own class.
  • protected: The member is accessible within its own class and by derived classes.
  • public: The member is accessible from any other class.
  • internal: The member is accessible within the same assembly.

2. Benefits of Encapsulation

  • Improves code maintainability and flexibility.
  • Protects the integrity of the data by restricting access to the internal state.
  • Encourages the use of public methods to interact with the data, which can be controlled and validated.

3. Example of Encapsulation

Here is an example of encapsulation in C# using a class with private fields and public properties:


    public class Person
    {
        // Private fields
        private string name;
        private int age;

        // Public property for 'Name'
        public string Name
        {
            get { return name; }
            set
            {
                if (!string.IsNullOrEmpty(value))
                {
                    name = value;
                }
            }
        }

        // Public property for 'Age'
        public int Age
        {
            get { return age; }
            set
            {
                if (value > 0)
                {
                    age = value;
                }
            }
        }

        // Method to display person details
        public void DisplayInfo()
        {
            Console.WriteLine($"Name: {Name}, Age: {Age}");
        }
    }

4. Using Encapsulation

Here's how you can use the encapsulated class in a program:


    class Program
    {
        static void Main()
        {
            // Create an instance of Person
            Person person = new Person();

            // Set properties
            person.Name = "John Doe";
            person.Age = 30;

            // Display information
            person.DisplayInfo(); // Output: Name: John Doe, Age: 30

            // Attempt to set invalid values
            person.Name = ""; // This will be ignored
            person.Age = -5;  // This will be ignored

            // Display information again
            person.DisplayInfo(); // Output: Name: John Doe, Age: 30
        }
    }

5. Key Points

  • Encapsulation bundles data and methods into a single class and restricts access to the internal state.
  • Access modifiers control the visibility and accessibility of class members.
  • Encapsulation helps in maintaining and protecting the integrity of data by exposing only necessary parts of an object.
