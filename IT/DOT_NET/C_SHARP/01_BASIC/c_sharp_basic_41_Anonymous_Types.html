C# Anonymous Types

Anonymous types in C# provide a convenient way to create objects without explicitly defining a class. They are useful for encapsulating a set of read-only properties into a single object.

1. Purpose

Anonymous types are used to:

  • Quickly Group Data: Create objects that group data without needing to define a new class.
  • Simplify Code: Reduce boilerplate code when you only need a temporary object.
  • Enhance Readability: Make code more concise and easier to understand in scenarios like LINQ queries.

2. Creating Anonymous Types

Anonymous types are created using the new keyword and an object initializer. You define the properties directly in the initializer, and the compiler generates a class with those properties.


    var person = new
    {
        Name = "Alice",
        Age = 30,
        IsEmployed = true
    };

    Console.WriteLine($"Name: {person.Name}");
    Console.WriteLine($"Age: {person.Age}");
    Console.WriteLine($"Employed: {person.IsEmployed}");

3. Properties of Anonymous Types

Anonymous types have the following characteristics:

  • Read-Only Properties: Properties of anonymous types are read-only.
  • Equality: Two anonymous type instances are considered equal if they have the same property names and values.
  • ToString Method: The ToString() method provides a string representation of the anonymous type.

4. Example Usage with LINQ

Anonymous types are commonly used with LINQ queries to project data into a new shape. Here's an example:


    var people = new[]
    {
        new { Name = "Alice", Age = 30 },
        new { Name = "Bob", Age = 25 }
    };

    var result = from person in people
                 select new
                 {
                     FullName = person.Name,
                     IsAdult = person.Age >= 18
                 };

    foreach (var item in result)
    {
        Console.WriteLine($"Full Name: {item.FullName}, Is Adult: {item.IsAdult}");
    }

5. Limitations

Anonymous types have some limitations:

  • No Access Modifiers: You cannot specify access modifiers for properties.
  • No Methods: Anonymous types cannot have methods or constructors.
  • Scope: They are limited to the scope where they are created and cannot be passed around easily.

6. Key Points

  • Anonymous Types are ideal for temporary data grouping without the need for class definitions.
  • Read-Only Properties and default equality checks are fundamental aspects of anonymous types.
  • LINQ Queries often utilize anonymous types for projecting data in a flexible manner.
  • Limitations include the inability to define methods and access modifiers, and they are confined to the local scope.
