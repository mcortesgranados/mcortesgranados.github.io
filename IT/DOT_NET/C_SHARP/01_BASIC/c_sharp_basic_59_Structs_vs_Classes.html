C# Structs vs Classes

In C#, both structs and classes are used to define types, but they have different characteristics and are used in different scenarios. Understanding the differences between them is important for making the right design choices in your application.

1. Overview

Structs: Value types that are typically used for small, lightweight data structures. They are stored on the stack and are copied by value.

Classes: Reference types that are used for larger, more complex data structures. They are stored on the heap and are copied by reference.

2. Memory Allocation

Structs: Allocated on the stack. This means they are generally faster to allocate and deallocate, but they have a limited lifetime and cannot have a dynamic size.

Classes: Allocated on the heap. This means they can have a larger and dynamic size, but memory management is handled by the garbage collector, which can introduce some overhead.

3. Default Constructor

Structs: Cannot have an explicit parameterless constructor. The default constructor initializes all fields to their default values.

Classes: Can have explicit parameterless constructors as well as constructors with parameters.

4. Inheritance

Structs: Do not support inheritance. They cannot derive from another struct or class and cannot be the base of another struct or class.

Classes: Support inheritance. They can derive from other classes and be the base of other classes.

5. Nullability

Structs: Cannot be null. They are value types and always have a default value even if not explicitly initialized.

Classes: Can be null. They are reference types and can be assigned a null value.

6. Examples

Struct Example


public struct Point
{
    public int X { get; set; }
    public int Y { get; set; }

    public Point(int x, int y)
    {
        X = x;
        Y = y;
    }
}

The Point struct represents a point in a 2D coordinate system. It is a value type and cannot be inherited from.

Class Example


public class Person
{
    public string Name { get; set; }
    public int Age { get; set; }

    public Person(string name, int age)
    {
        Name = name;
        Age = age;
    }
}

The Person class represents a person with a name and age. It is a reference type and can be inherited from.

7. Key Points

  • Memory Allocation: Structs are allocated on the stack, while classes are allocated on the heap.
  • Default Constructor: Structs cannot have explicit parameterless constructors; classes can.
  • Inheritance: Structs do not support inheritance; classes do.
  • Nullability: Structs cannot be null; classes can be null.
