C# Abstract Classes

An abstract class in C# is a class that cannot be instantiated directly. It is designed to be a base class from which other classes can derive. Abstract classes can contain abstract methods (methods without implementation) that derived classes must implement.

1. Declaring an Abstract Class

To declare a class as abstract, use the abstract keyword. Abstract classes can have both abstract methods and methods with implementations:


public abstract class Animal
{
    // Abstract method (does not have a body)
    public abstract void MakeSound();
    
    // Regular method (has a body)
    public void Sleep()
    {
        Console.WriteLine("Sleeping...");
    }
}

2. Implementing Abstract Methods

Any class that inherits from an abstract class must implement all abstract methods. Hereâ€™s an example:


public class Dog : Animal
{
    // Implementing the abstract method
    public override void MakeSound()
    {
        Console.WriteLine("Bark!");
    }
}

public class Cat : Animal
{
    // Implementing the abstract method
    public override void MakeSound()
    {
        Console.WriteLine("Meow!");
    }
}

In this example, both Dog and Cat classes inherit from Animal and provide their own implementations for the MakeSound method.

3. Abstract Class Usage

Abstract classes are used to define a common base with shared functionality and a common interface. They are particularly useful when you want to define a template for a group of related classes:


public class Program
{
    public static void Main()
    {
        Animal myDog = new Dog();
        Animal myCat = new Cat();
        
        myDog.MakeSound();  // Output: Bark!
        myCat.MakeSound();  // Output: Meow!
        
        myDog.Sleep();      // Output: Sleeping...
        myCat.Sleep();      // Output: Sleeping...
    }
}

4. Key Points

  • An abstract class cannot be instantiated directly.
  • Abstract methods in an abstract class must be implemented by derived classes.
  • Abstract classes can contain both abstract methods and methods with implementations.
  • Abstract classes are useful for defining a common base and enforcing a contract for derived classes.

Conclusion

Abstract classes in C# provide a way to define a common base class with shared functionality and a contract for derived classes. By using abstract classes, you can create a more structured and maintainable codebase that adheres to object-oriented principles.
