C# Indexers Usage

Indexers in C# allow objects to be indexed in a similar way to arrays. They provide a way to access elements of an object using an index, making the object act like an array or a list.

1. Basics of Indexers

An indexer is defined with the this keyword and provides access to the elements of an object using an index. Unlike properties, indexers can take parameters.

2. Example of Indexer Usage

Here's an example of defining and using an indexer in a custom class:


public class StringCollection
{
    private string[] items = new string[10];

    // Indexer definition
    public string this[int index]
    {
        get
        {
            if (index >= 0 && index < items.Length)
                return items[index];
            else
                throw new IndexOutOfRangeException("Index out of range");
        }
        set
        {
            if (index >= 0 && index < items.Length)
                items[index] = value;
            else
                throw new IndexOutOfRangeException("Index out of range");
        }
    }
}

public class Program
{
    public static void Main()
    {
        StringCollection collection = new StringCollection();

        // Using the indexer to set values
        collection[0] = "Hello";
        collection[1] = "World";

        // Using the indexer to get values
        Console.WriteLine(collection[0]);  // Output: Hello
        Console.WriteLine(collection[1]);  // Output: World
    }
}

3. Key Points

  • Indexer Definition: Use the this keyword followed by square brackets to define an indexer.
  • Accessing Elements: You can access elements using the defined indexer syntax, similar to array indexing.
  • Exception Handling: Handle exceptions for invalid index access to ensure robustness.

4. Conclusion

Indexers in C# provide a flexible way to access and manipulate data within an object using an index. They enhance the usability of custom classes by allowing them to be accessed similarly to arrays or collections.
