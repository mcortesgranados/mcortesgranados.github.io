C# Lambda Expressions

Lambda expressions in C# provide a concise way to write anonymous methods. They are used to create delegates or expression tree types in a more readable and compact syntax.

1. Purpose

Lambda expressions are used for:

  • Short-Circuiting Code: Writing small snippets of code inline without the need for separate method definitions.
  • Functional Programming: Enabling functional programming techniques, such as passing functions as arguments.
  • LINQ Queries: Providing a way to define query logic directly within LINQ expressions.

2. Syntax

The syntax of a lambda expression is:


    (parameters) => expression_or_statement_block

Where parameters are the input parameters (if any), and expression_or_statement_block is the code to execute.

3. Basic Examples

Here are some examples of lambda expressions:


// Lambda expression with one parameter and a single expression
Func square = x => x * x;
Console.WriteLine(square(5)); // Output: 25

// Lambda expression with multiple parameters and a block of code
Func add = (x, y) =>
{
    int result = x + y;
    return result;
};
Console.WriteLine(add(3, 4)); // Output: 7

5. Key Points

  • Lambda Expressions are concise, inline representations of anonymous methods.
  • Syntax involves parameters, the lambda operator (=>), and the expression or statement block.
  • Usage in LINQ allows for powerful and expressive data queries.
  • Types like Func and Action are often used with lambda expressions for functional programming.
