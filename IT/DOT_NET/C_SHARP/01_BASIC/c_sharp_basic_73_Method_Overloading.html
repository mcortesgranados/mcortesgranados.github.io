C# Method Overloading

Method overloading in C# allows you to define multiple methods with the same name but different parameter lists. This feature enables you to perform similar operations with different types or numbers of arguments using the same method name.

1. Basics of Method Overloading

To overload a method, you need to define multiple methods with the same name but different parameters. The method signature, which includes the method name and the parameter list, must be unique for each overloaded method.

2. Example of Method Overloading

Here's an example demonstrating method overloading with a method that calculates the area of different shapes:


public class Calculator
{
    // Method to calculate the area of a rectangle
    public int Area(int length, int width)
    {
        return length * width;
    }

    // Method to calculate the area of a square
    public int Area(int side)
    {
        return side * side;
    }

    // Method to calculate the area of a circle
    public double Area(double radius)
    {
        return Math.PI * radius * radius;
    }
}

3. Using Overloaded Methods

You can call overloaded methods based on the number and type of arguments passed. Here's how to use the Area methods defined above:


public class Program
{
    public static void Main()
    {
        Calculator calc = new Calculator();

        int rectangleArea = calc.Area(5, 10);
        Console.WriteLine("Area of rectangle: " + rectangleArea);

        int squareArea = calc.Area(4);
        Console.WriteLine("Area of square: " + squareArea);

        double circleArea = calc.Area(7.5);
        Console.WriteLine("Area of circle: " + circleArea);
    }
}

4. Key Points

  • Method overloading allows multiple methods with the same name but different parameter lists.
  • Each overloaded method must have a unique method signature (name and parameter list).
  • Overloaded methods can be differentiated by the number or type of parameters.

Conclusion

Method overloading in C# enhances code readability and usability by allowing multiple methods to perform similar operations with different arguments. This feature helps in reducing code duplication and making your code more intuitive.
