C# Implicit and Explicit Casts

Casting is the process of converting one type to another. In C#, there are two types of casting: implicit and explicit. Understanding when and how to use each type is important for handling type conversions in your programs.

1. Implicit Casts

Implicit casts are conversions that are automatically performed by the compiler. They are used when converting from a smaller to a larger data type, or when the conversion is guaranteed to be safe.

For example, converting an int to a double is implicit because a double can hold any value of an int without loss of information.


int number = 42;
double result = number; // Implicit cast from int to double

2. Explicit Casts

Explicit casts require the programmer to specify the conversion. They are used when converting between incompatible types or when there is a risk of data loss. Explicit casts are performed using a cast operator.

For example, converting a double to an int requires an explicit cast because the conversion may result in loss of data.


double pi = 3.14159;
int roundedPi = (int)pi; // Explicit cast from double to int

3. Custom Implicit and Explicit Casts

C# allows you to define custom implicit and explicit casts by overloading the cast operators in your own types. This can be useful when you need to define how your types should be converted.

Here's an example of defining custom casts:


public class Celsius
{
    public double Temperature { get; }

    public Celsius(double temperature)
    {
        Temperature = temperature;
    }

    // Implicit cast from Celsius to Fahrenheit
    public static implicit operator Fahrenheit(Celsius celsius)
    {
        return new Fahrenheit(celsius.Temperature * 9 / 5 + 32);
    }
}

public class Fahrenheit
{
    public double Temperature { get; }

    public Fahrenheit(double temperature)
    {
        Temperature = temperature;
    }

    // Explicit cast from Fahrenheit to Celsius
    public static explicit operator Celsius(Fahrenheit fahrenheit)
    {
        return new Celsius((fahrenheit.Temperature - 32) * 5 / 9);
    }
}

In this example, an implicit cast converts a Celsius object to a Fahrenheit object, and an explicit cast converts a Fahrenheit object to a Celsius object.

4. Key Points

  • Implicit Casts: Automatically performed by the compiler when converting from a smaller to a larger data type or when the conversion is safe.
  • Explicit Casts: Require the programmer to specify the conversion, used when there is a risk of data loss or when converting between incompatible types.
  • Custom Casts: Can be defined by overloading the cast operators in your own types to specify how conversions should be handled.
