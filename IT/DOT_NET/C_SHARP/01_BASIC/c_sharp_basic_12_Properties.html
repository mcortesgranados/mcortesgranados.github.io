C# Properties

In C#, properties provide a way to expose data members of a class while keeping the data encapsulated. They allow you to control access to the data with get and set accessors.

1. Defining a Property

A property is defined with a get accessor and an optional set accessor. The get accessor returns the property value, and the set accessor assigns a value to the property.


    public class Person
    {
        // Private fields
        private string name;
        private int age;

        // Public property for Name
        public string Name
        {
            get { return name; }
            set { name = value; }
        }

        // Public property for Age
        public int Age
        {
            get { return age; }
            set
            {
                if (value > 0)
                    age = value;
                else
                    Console.WriteLine("Age must be positive.");
            }
        }
    }

2. Read-Only and Write-Only Properties

Properties can be read-only, write-only, or have both accessors. A read-only property has only a get accessor, while a write-only property has only a set accessor.


    public class Person
    {
        private string name;
        private int age;

        // Read-only property
        public string Name
        {
            get { return name; }
        }

        // Write-only property
        public int Age
        {
            set
            {
                if (value > 0)
                    age = value;
                else
                    Console.WriteLine("Age must be positive.");
            }
        }

        // Constructor
        public Person(string name, int age)
        {
            this.name = name;
            Age = age;
        }
    }

3. Auto-Implemented Properties

Auto-implemented properties simplify property declarations by providing a shorthand syntax for defining properties without explicitly declaring a backing field.


    public class Person
    {
        // Auto-implemented properties
        public string Name { get; set; }
        public int Age { get; set; }
    }

4. Property with Backing Field

Sometimes, you need to include additional logic in the property accessors. This is where backing fields come into play. A backing field is a private field used to store the property value.


    public class Person
    {
        private string name;
        private int age;

        public string Name
        {
            get { return name; }
            set
            {
                if (!string.IsNullOrWhiteSpace(value))
                    name = value;
                else
                    Console.WriteLine("Name cannot be empty.");
            }
        }

        public int Age
        {
            get { return age; }
            set
            {
                if (value > 0)
                    age = value;
                else
                    Console.WriteLine("Age must be positive.");
            }
        }
    }

Hereâ€™s a complete C# program demonstrating various types of properties:


    using System;

    public class Person
    {
        private string name;
        private int age;

        public string Name
        {
            get { return name; }
            set
            {
                if (!string.IsNullOrWhiteSpace(value))
                    name = value;
                else
                    Console.WriteLine("Name cannot be empty.");
            }
        }

        public int Age
        {
            get { return age; }
            set
            {
                if (value > 0)
                    age = value;
                else
                    Console.WriteLine("Age must be positive.");
            }
        }

        // Constructor
        public Person(string name, int age)
        {
            Name = name;
            Age = age;
        }

        // Method to introduce the person
        public void Introduce()
        {
            Console.WriteLine($"Hello, my name is {Name} and I am {Age} years old.");
        }
    }

    class Program
    {
        static void Main()
        {
            // Creating a Person object
            Person person = new Person("Alice", 30);
            person.Introduce();

            // Modifying and accessing properties
            person.Name = "Bob";
            person.Age = 35;
            Console.WriteLine($"Updated Name: {person.Name}");
            Console.WriteLine($"Updated Age: {person.Age}");
        }
    }

Explanation

In the example program:

  • The Person class includes properties with get and set accessors, a read-only property, a write-only property, and auto-implemented properties.
  • The Main method demonstrates creating an object, using the properties, and calling a method that utilizes the properties.
