C# Sealed Classes

A sealed class in C# is a class that cannot be inherited. This means that no other class can derive from a sealed class. Sealed classes are used to prevent further inheritance, which can be useful in various scenarios to ensure the integrity of the class's implementation or to prevent misuse.

1. Declaring a Sealed Class

To declare a class as sealed, use the sealed keyword before the class keyword. Here is an example:


public sealed class FinalClass
{
    public void Display()
    {
        Console.WriteLine("This is a sealed class.");
    }
}

2. Attempting to Inherit a Sealed Class

Trying to inherit from a sealed class will result in a compile-time error. For example:


public class DerivedClass : FinalClass
{
    // This will cause a compile-time error.
}

The above code will not compile because FinalClass is sealed and cannot be inherited.

3. Use Cases for Sealed Classes

  • Performance Optimization: Sealing a class can potentially improve performance by allowing the runtime to make optimizations related to method calls and memory layout.
  • Preventing Extension: To ensure that a class's behavior remains unchanged and to prevent other developers from extending or modifying its behavior.
  • Security: To prevent creating subclasses that might introduce security vulnerabilities or inconsistencies.

4. Example Usage

Here is an example of how a sealed class might be used:


public sealed class Calculator
{
    public int Add(int a, int b)
    {
        return a + b;
    }
}

// Usage
public class Program
{
    public static void Main()
    {
        Calculator calc = new Calculator();
        Console.WriteLine("Sum: " + calc.Add(5, 10));
    }
}

In this example, the Calculator class is sealed, so it cannot be subclassed. The main purpose of this class is to provide a simple addition operation, ensuring that the implementation remains as intended without any extension.

5. Key Points

  • A sealed class cannot be inherited.
  • Use the sealed keyword to declare a sealed class.
  • Sealed classes are useful for performance optimization, security, and to prevent unwanted inheritance.

Conclusion

Sealed classes provide a way to control inheritance and ensure that the class's implementation remains secure and unaltered. By understanding and utilizing sealed classes appropriately, you can create more robust and maintainable code.
