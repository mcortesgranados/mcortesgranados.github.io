C# Generic Classes

Generic classes in C# allow you to define a class with a placeholder for the type of data it stores. This enables you to create reusable code that can work with different data types while maintaining type safety.

1. Declaring a Generic Class

To declare a generic class, use angle brackets (<>) to define type parameters. Here’s an example of a generic class that stores a value of any type:


public class Box<T>
{
    private T value;

    public void SetValue(T value)
    {
        this.value = value;
    }

    public T GetValue()
    {
        return value;
    }
}

2. Using a Generic Class

You can create instances of a generic class with different types. Here’s how to use the Box<T> class:


public class Program
{
    public static void Main()
    {
        // Using Box with an int
        Box<int> intBox = new Box<int>();
        intBox.SetValue(123);
        Console.WriteLine("Integer value: " + intBox.GetValue());

        // Using Box with a string
        Box<string> strBox = new Box<string>();
        strBox.SetValue("Hello, World!");
        Console.WriteLine("String value: " + strBox.GetValue());
    }
}

3. Generic Constraints

Sometimes you may want to restrict the types that can be used with a generic class. You can use generic constraints to specify such restrictions. For example, you can ensure that the type parameter implements a specific interface:


public class Repository<T> where T : IEntity
{
    public void Save(T entity)
    {
        // Save the entity to the database
    }
}

public interface IEntity
{
    int Id { get; set; }
}

In this example, the Repository<T> class has a constraint that requires T to implement the IEntity interface.

4. Key Points

  • Generic classes provide a way to define classes with placeholders for types.
  • Generic classes can be instantiated with different types, ensuring type safety.
  • Constraints can be used to restrict the types that can be used with a generic class.

Conclusion

Generic classes in C# are a powerful feature that allows you to create flexible and reusable code. By using generics, you can work with different data types while maintaining type safety and avoiding code duplication.
