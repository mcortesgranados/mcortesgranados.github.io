C# Properties and Accessors

In C#, properties are a special kind of class member that provide a flexible mechanism to read, write, or compute the values of private fields. Properties use accessors to control how values are retrieved and updated.

1. Purpose of Properties

Properties are used to:

  • Encapsulate Data: Provide controlled access to the data within a class.
  • Implement Data Validation: Ensure that the data being set is valid before storing it.
  • Provide Computed Values: Compute values on the fly based on other data within the class.

2. Defining Properties

A property consists of a get accessor, a set accessor, or both. The get accessor returns the property value, while the set accessor assigns a new value. Hereâ€™s a basic example:


public class Person
{
    private string name;

    public string Name
    {
        get { return name; }
        set { name = value; }
    }
}

In this example, the Name property has both get and set accessors, allowing for reading and writing the name field.

3. Read-Only Properties

Properties can be read-only by omitting the set accessor:


public class Person
{
    public string Name { get; }

    public Person(string name)
    {
        Name = name;
    }
}

In this example, the Name property is read-only after initialization, as there is no set accessor.

4. Write-Only Properties

Properties can also be write-only by omitting the get accessor:


public class Person
{
    private string name;

    public string Name
    {
        set { name = value; }
    }
}

In this example, the Name property can only be written to, not read from.

5. Auto-Implemented Properties

C# also supports auto-implemented properties, which simplify property definitions when no additional logic is required:


public class Person
{
    public string Name { get; set; }
}

In this example, C# automatically creates a private backing field for the Name property.

6. Computed Properties

Properties can also compute values dynamically based on other fields:


public class Rectangle
{
    public int Width { get; set; }
    public int Height { get; set; }

    public int Area
    {
        get { return Width * Height; }
    }
}

In this example, the Area property calculates the area of a rectangle based on its width and height.

7. Key Points

  • Encapsulation: Properties help to encapsulate data and provide controlled access.
  • Accessors: Use get to retrieve and set to assign values.
  • Read-Only and Write-Only: Properties can be read-only or write-only based on the accessors provided.
  • Auto-Implemented: Simplify property definitions with auto-implemented properties.
  • Computed Values: Use properties to compute values dynamically.
