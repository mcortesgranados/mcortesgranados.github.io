C# Exceptions

Exceptions in C# are a way to handle runtime errors and exceptional conditions that occur during the execution of a program. By using exceptions, you can manage errors gracefully and ensure your program can recover or fail in a controlled manner.

1. Exception Handling Basics

Exception handling in C# is done using the try, catch, finally, and throw keywords:

  • try: Defines a block of code that might throw an exception.
  • catch: Catches and handles the exception thrown by the try block.
  • finally: (Optional) Executes code after the try and catch blocks, regardless of whether an exception was thrown or not.
  • throw: Used to explicitly throw an exception.

    class Program
    {
        static void Main()
        {
            try
            {
                // Code that may throw an exception
                int[] numbers = { 1, 2, 3 };
                Console.WriteLine(numbers[5]); // This will throw an exception
            }
            catch (IndexOutOfRangeException ex)
            {
                // Handling the exception
                Console.WriteLine("An error occurred: " + ex.Message);
            }
            finally
            {
                // This code always executes
                Console.WriteLine("Execution completed.");
            }
        }
    }

3. Throwing Exceptions

You can throw exceptions manually using the throw keyword. This is useful for validating conditions and ensuring your program behaves as expected.


    class Program
    {
        static void ValidateAge(int age)
        {
            if (age < 0)
            {
                throw new ArgumentException("Age cannot be negative.");
            }
        }

        static void Main()
        {
            try
            {
                ValidateAge(-1); // This will throw an exception
            }
            catch (ArgumentException ex)
            {
                Console.WriteLine("Validation error: " + ex.Message);
            }
        }
    }

4. Custom Exceptions

You can create your own custom exception classes by inheriting from the Exception class. This is useful for creating more meaningful and specific exceptions for your application.


    public class CustomException : Exception
    {
        public CustomException(string message) : base(message) { }
    }

    class Program
    {
        static void Main()
        {
            try
            {
                throw new CustomException("This is a custom exception.");
            }
            catch (CustomException ex)
            {
                Console.WriteLine("Custom exception caught: " + ex.Message);
            }
        }
    }

5. Key Points

  • Exceptions provide a way to handle errors and unusual conditions in a controlled manner.
  • Use try, catch, and finally blocks to manage exceptions and ensure clean-up operations are performed.
  • You can throw and catch exceptions to manage errors and validate conditions in your code.
  • Custom exceptions allow for more specific error handling and clearer error messages.
