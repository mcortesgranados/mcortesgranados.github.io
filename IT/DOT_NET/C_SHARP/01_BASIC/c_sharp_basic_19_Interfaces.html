C# Interfaces

In C#, an interface is a contract that defines a set of methods and properties that a class must implement. Interfaces are used to provide a common functionality across different classes, allowing for a more flexible and extensible design.

1. Defining an Interface

Interfaces are defined using the interface keyword. They can contain method signatures, properties, events, and indexers, but cannot contain any implementation.


    public interface IShape
    {
        double CalculateArea();
        double CalculatePerimeter();
    }

2. Implementing an Interface

Classes or structs that implement an interface must provide implementations for all the members of the interface. The class or struct uses the implements keyword to declare that it is implementing an interface.


    public class Rectangle : IShape
    {
        public double Width { get; set; }
        public double Height { get; set; }

        public double CalculateArea()
        {
            return Width * Height;
        }

        public double CalculatePerimeter()
        {
            return 2 * (Width + Height);
        }
    }

    public class Circle : IShape
    {
        public double Radius { get; set; }

        public double CalculateArea()
        {
            return Math.PI * Radius * Radius;
        }

        public double CalculatePerimeter()
        {
            return 2 * Math.PI * Radius;
        }
    }

3. Using Interfaces

Once an interface is implemented, you can use it to reference objects of different classes through the interface type. This allows for polymorphism and flexibility in your code.


    class Program
    {
        static void Main()
        {
            IShape rectangle = new Rectangle { Width = 5, Height = 10 };
            IShape circle = new Circle { Radius = 7 };

            Console.WriteLine($"Rectangle Area: {rectangle.CalculateArea()}");
            Console.WriteLine($"Rectangle Perimeter: {rectangle.CalculatePerimeter()}");
            Console.WriteLine($"Circle Area: {circle.CalculateArea()}");
            Console.WriteLine($"Circle Perimeter: {circle.CalculatePerimeter()}");
        }
    }

4. Key Points

  • Interfaces define a contract for what a class can do, without specifying how it does it.
  • Classes and structs that implement an interface must provide concrete implementations for all of its members.
  • Interfaces support polymorphism, allowing objects of different classes to be treated through a common interface.
  • Interfaces can be implemented by multiple classes, and a class can implement multiple interfaces.
