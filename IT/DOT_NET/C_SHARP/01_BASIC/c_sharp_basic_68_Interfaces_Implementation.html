C# Interface Implementation

An interface in C# defines a contract that classes or structs must adhere to. It specifies what methods, properties, events, or indexers the implementing class must define, without specifying how they are implemented.

1. Defining an Interface

An interface is declared using the interface keyword. Here's a basic example:


public interface IAnimal
{
    void MakeSound();
    string Species { get; }
}

2. Implementing an Interface

A class or struct that implements an interface must provide concrete implementations for all its members. For example:


public class Dog : IAnimal
{
    public string Species { get { return "Dog"; } }
    
    public void MakeSound()
    {
        Console.WriteLine("Bark!");
    }
}

public class Cat : IAnimal
{
    public string Species { get { return "Cat"; } }
    
    public void MakeSound()
    {
        Console.WriteLine("Meow!");
    }
}

In this example, both Dog and Cat classes implement the IAnimal interface, providing their own versions of the MakeSound method and the Species property.

3. Using an Interface

Once a class implements an interface, you can use it as follows:


public class Program
{
    public static void Main()
    {
        IAnimal dog = new Dog();
        IAnimal cat = new Cat();
        
        dog.MakeSound();  // Output: Bark!
        cat.MakeSound();  // Output: Meow!
        
        Console.WriteLine(dog.Species);  // Output: Dog
        Console.WriteLine(cat.Species);  // Output: Cat
    }
}

4. Interface Inheritance

Interfaces can inherit from other interfaces, just like classes. When a class implements a derived interface, it must implement all the members of both the derived and base interfaces.


public interface IWalkable
{
    void Walk();
}

public interface IAnimalWithLegs : IAnimal, IWalkable
{
    int NumberOfLegs { get; }
}

public class Horse : IAnimalWithLegs
{
    public string Species { get { return "Horse"; } }
    
    public int NumberOfLegs { get { return 4; } }
    
    public void MakeSound()
    {
        Console.WriteLine("Neigh!");
    }

    public void Walk()
    {
        Console.WriteLine("Horse is walking.");
    }
}

In this example, the Horse class implements the IAnimalWithLegs interface, which inherits from both IAnimal and IWalkable.

5. Explicit Interface Implementation

In some cases, a class might need to implement members of multiple interfaces that have the same signature. To avoid ambiguity, you can use explicit interface implementation:


public interface IPrinter
{
    void Print();
}

public interface IScanner
{
    void Print();
}

public class MultiFunctionDevice : IPrinter, IScanner
{
    // Explicit interface implementation
    void IPrinter.Print()
    {
        Console.WriteLine("Printing a document.");
    }

    void IScanner.Print()
    {
        Console.WriteLine("Scanning a document.");
    }
}

In this case, MultiFunctionDevice implements both IPrinter and IScanner, but the Print method from each interface is explicitly implemented to handle the respective functionality.

Conclusion

Interfaces are a powerful tool in C# for enforcing contracts and ensuring that classes adhere to specific patterns of behavior. They promote flexibility, abstraction, and help decouple components in your codebase.
