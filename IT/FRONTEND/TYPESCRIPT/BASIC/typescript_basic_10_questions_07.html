TypeScript Basic Level Questions and Answers - Part 7

1. How do you define a tuple in TypeScript?

A tuple is a fixed-length array with specified types for each element in TypeScript.

let tuple: [string, number];
tuple = ["hello", 42];
// tuple = [42, "hello"]; // Error: Type 'number' is not assignable to type 'string'

2. How do you define optional properties in an interface in TypeScript?

You can mark properties as optional using the question mark (?) after the property name.

interface Person {
    name: string;
    age?: number; // Optional property
}

let person: Person = { name: "Alice" };
console.log(person.age); // undefined (age is optional)

3. What is the void type in TypeScript?

The void type is used to indicate that a function does not return a value.

function logMessage(message: string): void {
    console.log(message);
}

logMessage("Hello, world!"); // Output: Hello, world!

4. How do you declare a union of string literals in TypeScript?

A union of string literals allows you to specify multiple possible values for a variable.

type Direction = "Up" | "Down" | "Left" | "Right";

let move: Direction;
move = "Up";   // Valid
// move = "Forward"; // Error: Type '"Forward"' is not assignable to type '"Up" | "Down" | "Left" | "Right"'

5. How do you define a class with a constructor in TypeScript?

A class constructor initializes the class properties when a new instance is created.

class Car {
    make: string;
    model: string;

    constructor(make: string, model: string) {
        this.make = make;
        this.model = model;
    }

    getDetails(): string {
        return `${this.make} ${this.model}`;
    }
}

let myCar = new Car("Toyota", "Corolla");
console.log(myCar.getDetails()); // Toyota Corolla

6. How do you use an interface to enforce function parameter structure?

You can use an interface to define the structure of objects passed as function arguments.

interface User {
    name: string;
    age: number;
}

function greetUser(user: User): string {
    return `Hello, ${user.name}!`;
}

const user = { name: "Alice", age: 25 };
console.log(greetUser(user)); // Hello, Alice!

7. How do you handle default values in function parameters in TypeScript?

Default values can be set for function parameters to ensure they have a value if none is provided.

function greet(name: string = "Guest"): string {
    return `Hello, ${name}!`;
}

console.log(greet()); // Hello, Guest!
console.log(greet("Alice")); // Hello, Alice!

8. How do you create a class with private members in TypeScript?

Private members can only be accessed within the class itself.

class Employee {
    private name: string;

    constructor(name: string) {
        this.name = name;
    }

    getName(): string {
        return this.name;
    }
}

let emp = new Employee("Bob");
console.log(emp.getName()); // Bob
// console.log(emp.name); // Error: Property 'name' is private

9. How do you use the readonly modifier for properties in TypeScript?

The readonly modifier ensures that a property can only be assigned once and cannot be changed afterward.

class Book {
    readonly title: string;

    constructor(title: string) {
        this.title = title;
    }
}

let book = new Book("TypeScript Basics");
// book.title = "New Title"; // Error: Cannot assign to 'title' because it is a read-only property

10. How do you create a function with rest parameters in TypeScript?

Rest parameters allow a function to accept an indefinite number of arguments as an array.

function sum(...numbers: number[]): number {
    return numbers.reduce((total, num) => total + num, 0);
}

console.log(sum(1, 2, 3)); // 6
console.log(sum(4, 5, 6, 7)); // 22
