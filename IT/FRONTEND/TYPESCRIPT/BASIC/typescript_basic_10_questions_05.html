TypeScript Basic Level Questions and Answers - Part 5

1. What is a never type in TypeScript?

The never type represents values that never occur, often used in functions that throw errors or infinite loops.

function throwError(message: string): never {
    throw new Error(message);
}
// No value is returned, the function crashes

2. How do you use the Partial utility type in TypeScript?

The Partial type makes all properties of a type optional.

interface User {
    name: string;
    age: number;
}

let partialUser: Partial = {
    name: "Alice"
};

3. How do you use the Required utility type in TypeScript?

The Required type makes all properties of a type required.

interface User {
    name?: string;
    age?: number;
}

let user: Required = {
    name: "John",
    age: 30
};

4. How do you define default parameters in TypeScript functions?

You can set default values for function parameters to ensure they have a value if none is provided.

function greet(name: string = "Guest"): string {
    return `Hello, ${name}!`;
}

console.log(greet()); // Hello, Guest!
console.log(greet("Alice")); // Hello, Alice!

5. What is the difference between an interface and a type alias in TypeScript?

An interface is a blueprint for an object, while a type alias can represent primitives, objects, or even unions.

// Interface
interface Person {
    name: string;
    age: number;
}

// Type alias
type StringOrNumber = string | number;

6. How do you handle function overloading in TypeScript?

Function overloading allows you to define multiple function signatures for a single function.

function combine(a: string, b: string): string;
function combine(a: number, b: number): number;
function combine(a: any, b: any): any {
    return a + b;
}

console.log(combine(1, 2)); // 3
console.log(combine("Hello, ", "world!")); // Hello, world!

7. How do you declare a constant enum in TypeScript?

Constant enums are compiled to inline values and do not include additional code in the JavaScript output.

const enum Directions {
    Up,
    Down,
    Left,
    Right
}

let direction: Directions = Directions.Up;
console.log(direction); // 0 (compiled value)

8. What is an abstract class in TypeScript?

An abstract class is a class that cannot be instantiated directly. It is used as a base class for other classes.

abstract class Animal {
    abstract makeSound(): void;

    move(): void {
        console.log("Moving...");
    }
}

class Dog extends Animal {
    makeSound(): void {
        console.log("Woof!");
    }
}

const dog = new Dog();
dog.makeSound(); // Woof!

9. How do you create a type-safe keyof operator for objects?

TypeScript's keyof operator can be used to get the keys of an object in a type-safe way.

interface Car {
    model: string;
    year: number;
}

type CarKeys = keyof Car; // "model" | "year"

let key: CarKeys = "model"; // Valid
// key = "color"; // Error: Type '"color"' is not assignable to type '"model" | "year"'

10. How do you use the NonNullable utility type in TypeScript?

The NonNullable type removes null and undefined from a type.

type NullableString = string | null | undefined;
type NonNullableString = NonNullable;

let str: NonNullableString = "Hello";
// str = null; // Error: Type 'null' is not assignable to type 'string'
