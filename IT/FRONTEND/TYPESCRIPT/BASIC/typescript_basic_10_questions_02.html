TypeScript Basic Level Questions and Answers - Part 2

1. What is a type alias in TypeScript?

A type alias is used to create a new name for a type. It can be used to simplify complex types.

type ID = number | string;

let userId: ID = 123;
userId = "abc";

2. How do you use TypeScript with arrays?

Arrays in TypeScript can be typed to ensure that only elements of a specific type are allowed.

let numbers: number[] = [1, 2, 3, 4];
let names: Array = ["Alice", "Bob", "Charlie"];

3. What is an enum in TypeScript?

An enum is a way to give names to a set of numeric or string values, making code more readable.

enum Direction {
    Up,
    Down,
    Left,
    Right
}

let move: Direction = Direction.Up;

4. How do you handle default parameters in TypeScript?

Default parameters can be assigned a value if no argument is provided.

function greet(name: string = "Guest"): string {
    return `Hello, ${name}`;
}

console.log(greet()); // Hello, Guest
console.log(greet("John")); // Hello, John

5. What is the never type in TypeScript?

The never type represents values that never occur. It is often used for functions that never return.

function throwError(message: string): never {
    throw new Error(message);
}

6. How do you use the Readonly utility type in TypeScript?

The Readonly utility type creates a type where all properties are read-only.

type User = {
    name: string;
    age: number;
}

const user: Readonly = {
    name: "Alice",
    age: 25
};

// user.age = 26; // Error: Cannot assign to 'age' because it is a read-only property

7. How do you create a type with optional properties?

Optional properties are denoted with a question mark ? after the property name.

interface Product {
    name: string;
    price?: number;
}

const product1: Product = { name: "Laptop" };
const product2: Product = { name: "Phone", price: 699 };

8. How do you use the Partial utility type in TypeScript?

The Partial utility type makes all properties of a type optional.

interface User {
    name: string;
    age: number;
}

const updateUser: Partial = {
    age: 30
};

9. What are TypeScript decorators?

Decorators are special functions that can be attached to classes, methods, properties, or parameters. They are used to add metadata or modify behavior.

@decorator
class MyClass {
    @propertyDecorator
    myProperty: string;
}

function decorator(target: Function) {
    console.log("Decorator called");
}

function propertyDecorator(target: any, propertyKey: string) {
    console.log(`Property decorator called for ${propertyKey}`);
}

10. How do you handle asynchronous code in TypeScript?

TypeScript supports async/await syntax for handling asynchronous operations.

async function fetchData(url: string): Promise {
    const response = await fetch(url);
    const data = await response.text();
    return data;
}

fetchData("https://api.example.com/data")
    .then(data => console.log(data))
    .catch(error => console.error(error));
